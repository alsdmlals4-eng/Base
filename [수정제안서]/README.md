# Base 수정제안서

다른 프로젝트에서 발견한 교훈을 Base 공용 규칙으로 승격하기 전에 보존·검토하는 대기 영역이다. 이 폴더의 제안은 활성 Base 계약이 아니며 사용자 승인 전에는 Method·Skill·Template·Tool·Test를 변경하지 않는다.

## 제출 흐름

```text
프로젝트 증거와 반례 수집
→ 프로젝트 고유 정보 제거
→ BCP 제안서와 Registry 항목 작성
→ 제안 전용 PR
→ 사용자 검토
→ APPROVED_FOR_IMPLEMENTATION
→ 별도 구현 PR
→ 검증·병합 후 IMPLEMENTED
```

- 신규 제안 PR은 `[수정제안서]/**`만 변경한다.
- 구현 PR은 기존 제안의 `approval_ref`와 승인 범위를 제시한다.
- 사용자가 승인한 직접 요청은 별도 제안서 없이 작업 계약이 될 수 있다.
- 제안과 증거는 거절·보류·구현 후에도 삭제하지 않는다.
- 실행 파일·폰트·입력 자료·계정 인증·저장소 권한이 필요하면 이유, 설치·적용 방법, 확인 명령과 최소 권한을 사용자에게 요청한다.

## 제안서 필수 내용

- 출처 프로젝트·기준 커밋·제출일
- 관찰 사실과 재현 증거
- 공용화할 원리와 프로젝트 전용으로 남길 내용
- 적용 조건·비사용 조건·반례
- 영향 경로·호환성·보안·비용 위험
- 검증 시나리오와 롤백
- 사용자 승인 근거와 구현 PR

상태 책임 원본은 `PROPOSAL_REGISTRY.json`이다.
