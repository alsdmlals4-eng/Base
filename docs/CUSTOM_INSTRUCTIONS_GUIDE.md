# Custom Instructions Guide

이 문서는 ChatGPT, Codex, Copilot 같은 AI 도구의 맞춤형 지침을 작성할 때 참고하는 공용 기준이다.

## 1. 핵심 원칙

맞춤형 지침은 짧고 안정적인 행동 기준만 담는다.

- 사용자의 최신 요청과 의도 파악을 최우선으로 둔다.
- 프로젝트별 실제 작업은 Base 원격이 아니라 프로젝트 내부 로컬 사본을 먼저 읽는다.
- 긴 프로젝트 세계관, 특정 파일명, 현재 MVP 세부 구현은 맞춤형 지침이 아니라 프로젝트 저장소 문서에 둔다.
- 불확실하면 추정하지 말고 확인한다.
- 사용자의 기존 변경사항을 보호한다.
- 작업 범위 밖 기능 추가와 불필요한 리팩터링을 피한다.

## 2. ChatGPT 맞춤형 지침 기준

ChatGPT용 지침은 다음 역할을 강화한다.

- 사용자의 요청을 바로 처리하지 않고 의도와 목표를 먼저 파악한다.
- 짧은 요청을 좋은 프롬프트로 변환한다.
- 기획, 벤치마킹, 시스템 설계, Issue, Codex Goal, HTML 대시보드, 검토, 테스트 체크리스트를 담당한다.
- 실제 코드 구현 완료를 추정하지 않는다.
- 작업 결과가 Base 승격 후보인지 프로젝트 전용인지 구분한다.

## 3. Codex 맞춤형 지침 기준

Codex용 지침은 다음 역할을 강화한다.

- Codex는 구현 담당자다.
- 작업 전 프로젝트의 `AGENTS.md`, `docs/BASE_RULES_VERSION.md`, Base 로컬 사본, 프로젝트 전용 규칙, 현재 Issue/Goal, 실제 수정 대상 파일을 읽는다.
- 파일을 수정하기 전에 이해한 내용, 가정, 수정 대상, 검증 계획을 짧게 보고한다.
- 사라지거나 이름이 바뀌거나 크게 수정된 파일은 변경 이유와 다른 파일에 미치는 영향을 보고한다.
- 사용자의 기존 변경사항을 되돌리거나 덮어쓰지 않는다.
- 정상 작동하는 구조를 취향이나 미관만으로 바꾸지 않는다.
- 작업 후 변경 파일, 변경 이유, 구현 내용, 검증 내용, 남은 위험, Base 승격 후보를 보고한다.

## 4. 파일 변경 설명 규칙

AI가 파일을 생성, 수정, 삭제, 이동, 이름 변경할 때는 다음을 남긴다.

```md
## 파일 변경 의도

- 파일:
- 변경 유형: 생성 / 수정 / 삭제 / 이동 / 이름 변경
- 변경 이유:
- 연결되는 문서 또는 코드:
- 다른 AI 또는 다음 작업자가 알아야 할 영향:
- 후속 동기화 필요 여부:
```

삭제나 이름 변경은 충돌 위험이 크므로, 가능한 경우 먼저 대체 경로와 참조 업데이트 여부를 확인한다.

## 5. 지침을 너무 길게 만들지 않는 기준

맞춤형 지침에는 다음을 넣지 않는다.

- 현재 프로젝트의 긴 세계관
- 특정 MVP의 전체 명세
- 긴 벤치마킹 목록
- 구체적인 코드 파일 전체 목록
- 일회성 작업 지시

이 내용들은 GitHub Issue, PROJECT_BRIEF, DESIGN_INTENT, README, TEST_CHECKLIST, 프로젝트 전용 Codex 규칙에 둔다.
