# AI Shared Work Rules

이 문서는 특정 엔진이나 장르에 종속되지 않는 공용 AI 실행 규칙만 책임진다.

- 전체 운영 구조와 상태·발행 정책: `docs/OPERATING_MODEL.md`
- 요청 라우팅·요구 확인·실행 계약: `skills/managing-project-intake-and-work-contract/SKILL.md`
- 작업·제품 게이트: `docs/knowledge/methods/DEVELOPMENT_GATES_METHOD.md`
- 작업별 최소 읽기: `docs/DOCUMENTATION_MAP.md`

같은 작업 생명주기와 체크리스트를 이 문서에 다시 복제하지 않는다.

## 1. 규칙 우선순위

1. 사용자의 최신 지시
2. 프로젝트 루트 `AGENTS.md`와 보안·엔진·데이터 규칙
3. 프로젝트 Active Context와 승인된 작업 계약
4. 등록된 책임 원본과 실제 파일·테스트
5. 프로젝트에 동기화된 Base 기준
6. Base 원격 원본
7. 과거 대화와 추측

정상 동작 중인 사용자 변경사항을 되돌리지 않는다.

## 2. 역할 분리

- 기획·조정 AI: 사용자 의도, 플레이어 가치, 작업 계약, 검수 기준을 정리한다.
- 구현 AI: 승인된 범위에서 실제 파일을 수정하고 검증한다.
- 사용자: 방향·비용·위험을 결정하고 실제 결과와 플레이 감각을 확인한다.

기획 담당은 구현 완료를 추정하지 않고, 구현 담당은 제품 방향을 임의 확장하지 않는다.

## 3. 공용·전용 경계

### Base — 공용

- 반복 가능한 판단·작업·검수 방법
- 실행 Skill과 Template
- 성공·실패·미검증 Case
- 자동화 가능한 Schema·Checker·Test

### 프로젝트 — 전용

- 세계관·캐릭터·수치·규칙
- 엔진·데이터·코드·씬·자산과 실제 경로
- 승인 이미지·프롬프트·비용·계정
- 구현·검증 상태와 프로젝트 Skill 확장

프로젝트의 구체 해답을 Base에 복사하지 않는다.

## 4. 도구·권한·보안

- 필요한 실행 파일·라이브러리·폰트·입력 파일·인증·권한을 작업과 검증 전에 확인한다.
- 누락 항목은 필요한 이유, 공식 설치·적용 방법, 확인 명령과 최소 권한을 안내한다.
- 사용자 승인 없이 시스템 전역 설치, 권한 확대, 계정·브랜치 보호 설정을 변경하지 않는다.
- 비밀값·비공개 원문·권한 없는 자료를 출력·저장·외부 전송하지 않는다.
- 외부 스크립트·hook·MCP·모델 전송은 목적·범위·복구 가능성을 먼저 검토한다.
- 설치되지 않은 도구를 사용한 것처럼 처리하거나 실행하지 않은 검증을 통과로 보고하지 않는다.

## 5. 구현·변경 안전

- 승인 범위를 만족하는 가장 작은 검증 가능한 변경을 사용한다.
- 기능 추가와 대규모 리팩터링을 가능한 한 분리한다.
- 저장 형식·공개 인터페이스·사용자 흐름·승인 자산을 보호한다.
- `[보류]`는 재개 승인 없이 구현하지 않는다.
- 범위 밖 개선은 별도 작업 계약이나 제안으로 분리한다.
- 기존 프로젝트 구조 변경은 `managing-game-project-operating-system`의 `audit`부터 시작한다.

## 6. 검증과 완료 보고

프로젝트에 실제로 존재하는 검증만 실행한다.

```text
포맷·문법·정적 검사
→ 관련 자동 테스트
→ 핵심 정상·실패·경계 경로
→ 저장·호환성
→ 실제 화면·플레이·오디오·성능
→ diff와 승인 범위
→ 사용자 수동 검수
```

완료 보고는 `실제 변경`, `검증 완료`, `미검증`, `사용자 확인 대기`, `남은 위험·롤백`을 분리한다.

## 7. 조사·벤치마킹

- 현재 결정에 필요한 범위만 공식·1차 출처 중심으로 조사한다.
- 사례 수보다 채택·변형·제외 결론을 우선한다.
- 다른 작품의 코드·아트·문구·UI를 그대로 복제하지 않는다.
- 이미 승인된 작은 구현에 벤치마킹을 새로 강제하지 않는다.

## 8. 학습 환류

실패, 중요한 결정, 재사용 가능한 교훈, 실제 검증 결과가 있는 Skill 호출만 Learning Log에 기록한다. 한 번의 성공은 관찰 또는 가설이다.

프로젝트 교훈의 공용화는 `skills/managing-base-change-proposals/SKILL.md`를 사용하며, 제안 PR과 승인된 구현 PR을 분리한다.
