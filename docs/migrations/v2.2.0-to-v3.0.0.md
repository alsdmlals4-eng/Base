# Base v2.2.0 → v3.0.0 수동 마이그레이션

schema v3는 적용 프로젝트가 없는 시점에 JSON 전용 계약을 즉시 교체한다. v2 자동 호환 분기는 제공하지 않으며, v2 Registry나 Manifest를 읽으면 생성기와 검사기는 명확한 버전 오류를 낸다.

## 계약 변경

| v2.2.0 | v3.0.0 |
|---|---|
| `source_json` | `source_path` + `source_format: markdown | json` |
| 모든 기획 본책 JSON | 서술은 Markdown 기본, 구조 데이터는 JSON |
| DOCX 필수 출력 | `output_docx` 선택 또는 `null` |
| 고정 생성 다이어그램 | `diagram_policy: none | mermaid | generated` |
| Markdown 기획서 금지 | Registry에 등록한 Markdown 허용 |
| Skill Map Markdown 금지 | Registry에서 선택적으로 자동 생성하는 파생본 허용 |
| `status: CURRENT` | `sync_status: CURRENT` |
| 사람 검수와 최신성 혼재 | `sync_status`, 자동 렌더, 사람 검수를 독립 상태로 관리 |

## 수동 절차

1. 기존 Registry, 책임 원본, 승인 이미지와 Manifest를 백업이 아닌 Git 커밋으로 보존한다.
2. 각 문서의 성격을 판단해 단일 `source_path`와 `source_format`을 선택한다.
3. 같은 서술이 Markdown과 JSON 양쪽에 있으면 한쪽만 책임 원본으로 남기고 다른 쪽은 Registry 경로만 참조한다.
4. 서술형 Markdown은 필수 H1/H2 계약을 맞춘다.
5. 구조형 JSON은 `structured-design-document-v3.schema.json`에 맞춘다.
6. DOCX가 Word 검토에 필요하지 않으면 `output_docx: null`로 바꾼다.
7. 필요한 문서에만 Mermaid 또는 생성 다이어그램 정책을 지정한다.
8. 기존 v2 Manifest를 재사용하지 말고 v3 생성기로 PDF와 Manifest를 새로 생성한다.
9. 전 페이지 자동 렌더와 Codex 시각 검수를 수행한다. 사용자가 보지 않았다면 `human_visual_review: NOT_RUN`을 유지한다.
10. Governance와 동일 입력 재실행 diff 0을 확인한다.

## 실행

```powershell
python -m pip install -r requirements-publication.txt
pnpm install --frozen-lockfile
python tools/check_publication_environment.py --require-mermaid
python tools/build_design_documents.py --registry "[기획서]/00_프로젝트_허브/DESIGN_DOCUMENT_REGISTRY.json" --source-commit HEAD
```

실패하면 기존 정상 산출물은 보존된다. 환경 경로는 `BASE_LIBREOFFICE`, `BASE_PDFTOPPM`, `BASE_MERMAID_CLI`, `BASE_FONT_REGULAR`, `BASE_FONT_BOLD`, `PUPPETEER_EXECUTABLE_PATH`로 재정의한다.
