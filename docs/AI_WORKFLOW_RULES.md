# 공용 AI 작업 흐름

## 시작 전

1. 프로젝트의 `AGENTS.md`와 로컬 공용 규칙 버전을 확인한다.
2. 현재 Issue, 완료 기준, 제외 범위, 관련 파일을 확인한다.
3. 저장소 상태와 기존 사용자 변경사항을 확인한다.
4. 수정 대상, 데이터 흐름, 검증 방법을 짧게 정리한다.

## 구현 중

- 작업 범위 밖 변경은 승인 없이 하지 않는다.
- 변경 전 관련 호출 위치와 데이터 계약을 확인한다.
- 문서, 코드, 데이터의 설명이 실제 동작과 일치하게 유지한다.
- 불확실하거나 외부 상태에 의존하는 사실은 확인 가능한 근거를 남긴다.

## 종료 전

1. 정적 검사, 포맷, 테스트 또는 실행 가능한 검증을 수행한다.
2. 완료 기준을 항목별로 확인한다.
3. 미검증 항목과 외부 확인이 필요한 위험을 분리한다.
4. 기준 Base 커밋이 바뀌었다면 프로젝트의 버전 기록을 갱신한다.

## Goal 작성 기준

Goal에는 목표, 작업 전 확인 항목, 구현 범위, 제외 범위, 완료 기준, 검증 방법, 작업 후 보고 형식을 포함한다. 완료 기준은 체크리스트로 만들고, 구현 가능한 사실로 쓴다.
