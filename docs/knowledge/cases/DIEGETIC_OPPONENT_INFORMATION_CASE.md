# 사례 — 내부 난도·성장 데이터를 세계관 표현으로 학습시키기

- 출처 프로젝트: 십보강호
- 확인 날짜: 2026-07-16
- 작성 상태: 채택·구현 전 검증 필요
- 주제: 상대의 내부 강도·행운·성장 성향을 개발용 태그로 직접 노출하지 않고 일관된 세계관 문구로 암시

## 1. 문제

상대 선택 전에 위험도를 알려주면 전략 판단은 쉬워지지만 다음과 같은 개발용 표시는 세계관 몰입과 반복 학습을 약화할 수 있다.

```text
[강적] [A++] [핵심 능력 올인형]
```

반대로 정보를 모두 숨기면 제약·난도 선택이 추측에 가까워지고 불공정하게 느껴질 수 있다.

## 2. 맥락과 제약

- 상대는 내부적으로 등급, 행운, 성장 포인트, 투자 스타일과 종합 강도를 가진다.
- 플레이어는 상대 정보를 보고 추가 제약이나 위험을 선택한다.
- 처음에는 모든 표현의 정확한 뜻을 몰라도 반복 플레이로 의미를 학습해야 한다.
- 표시가 실제 내부 데이터와 모순되어서는 안 된다.
- 세계관 문구는 전투 계산을 변경하지 않는다.

## 3. 검토한 대안

### A. 개발용 태그 직접 공개

장점:

- 즉시 이해하기 쉽다.
- 비교와 밸런스 QA가 편하다.

단점:

- 상대가 인물보다 수치 묶음으로 보인다.
- 세계관 어휘를 학습할 이유가 없다.
- 플레이어 화면과 개발 화면의 경계가 사라진다.

### B. 모든 정보를 숨김

장점:

- 미지의 상대와 무림 소문이라는 분위기가 강하다.

단점:

- 제약 선택의 근거가 부족하다.
- 패배 원인을 상대 정보 부족으로만 돌릴 수 있다.
- 반복 플레이에서도 학습 가능한 규칙이 남지 않는다.

### C. 내부 의미 키를 일관된 세계관 표현으로 변환

장점:

- 전략 정보와 세계관 몰입을 함께 유지한다.
- 반복되는 어휘를 통해 사용자가 의미를 자연스럽게 학습한다.
- 정탐·조사 같은 정보 획득 시스템을 단계화할 수 있다.

채택: C.

## 4. 결정 구조

```text
내부 상대 프로필
- 종합 강도
- 성장 성과
- 우연·행운 보정
- 투자 스타일
- 실제 능력 경계

↓ 의미 키 매핑

플레이어 표현
- 이명·수식언
- 풍문·최근 전적
- 능력 평가 문장
- 정탐 단계별 구체화
```

표현 역할:

| 표현 | 암시 대상 |
|---|---|
| 이명·수식언 | 종합 강도와 대표 전법 |
| 풍문·최근 전적 | 성장세와 우연한 추가 성장 |
| 능력 평가 | 균형 투자, 한 계통 집중, 보조 능력 연계, 다계통 분산 |
| 정탐 | 같은 의미를 더 구체적인 문장으로 제공 |

## 5. 중요한 계약

- 같은 내부 의미에는 같은 어휘 계열을 반복한다.
- 문구는 무작위 장식이 아니라 실제 프로필 키와 연결한다.
- 강한 상대를 약한 표현으로 속이지 않는다.
- 정보 부족은 허용하지만 표시와 실제 데이터의 모순은 허용하지 않는다.
- 정탐은 반드시 원시 숫자를 공개할 필요가 없다. `최상위 기술의 문턱`, `내외를 함께 닦음`, `한 계통에 매진`처럼 표현의 구체성을 높일 수 있다.
- QA·디버그 화면에서는 내부 태그를 별도로 볼 수 있지만 기본 플레이 UI에는 노출하지 않는다.

## 6. 예상 가치

- 상대 정보 확인 자체가 세계관 경험이 된다.
- 반복 플레이가 어휘 학습과 위험 판단 숙련으로 이어진다.
- 제약·베팅·임무 선택에 필요한 공정한 근거를 제공한다.
- 내부 밸런스 데이터와 플레이어용 카피를 독립적으로 수정할 수 있다.

## 7. 위험

- 표현 종류가 너무 많으면 의미를 학습하지 못한다.
- 문구 작성자가 내부 데이터 계약을 모르고 분위기만 보고 문장을 만들 수 있다.
- 같은 어휘를 여러 의미에 사용하면 정보 신뢰가 무너진다.
- 현지화에서 강도와 성향의 어감 단계가 달라질 수 있다.
- 플레이어가 의도한 의미와 다른 방식으로 문구를 해석할 수 있다.

## 8. 검증 방법

- 처음 보는 사용자가 문구만 보고 상대 간 상대적 위험도를 정렬하는지 확인한다.
- 여러 회차 뒤 같은 어휘 계열의 의미를 설명하는지 확인한다.
- 상대 정보가 제약·난도 선택에 실제로 사용되는지 관찰한다.
- 표시 문구와 내부 프로필의 자동 매핑 불일치를 검사한다.
- 정탐 전·후 정보가 모순되지 않고 구체성만 증가하는지 확인한다.
- 현지화 언어별로 강도 단계와 투자 성향의 어감이 유지되는지 검수한다.

## 9. 재사용 가능한 원칙

- 내부 계산과 플레이어가 읽는 세계관 표현을 분리한다.
- 숨김과 직접 수치 공개 사이에 의미 기반 표현 계층을 둘 수 있다.
- 학습 가능한 표현은 반복성과 일관성이 핵심이다.
- 높은 정보 단계는 숫자 노출보다 표현의 구체화로 설계할 수 있다.
- 플레이어용 카피가 전투·경제 데이터를 소유하거나 수정하게 하지 않는다.

## 10. 그대로 복사하면 안 되는 요소

- 특정 프로젝트의 등급, 확률, 포인트와 성장 규칙.
- 무협식 이명·문파·무공 문장.
- 특정 상대 이름과 데이터 키.
- 제약·대회 구조.

정보를 직접 비교하고 최적화하는 것이 핵심인 분석 게임, 경쟁 PvP, 접근성상 정밀 수치가 필수인 화면에서는 세계관 표현만으로 대체하지 않는다. 이 경우 수치 보기 옵션이나 상세 분석 화면을 함께 제공한다.
