# 벤치마킹 사례 — 짧은 동료 반응과 학습 가능한 실패로 선택을 기억시키기

- 출처 유형: 공개 캐릭터·선택·규칙 학습 게임 벤치마킹
- 참고 사례: Disco Elysium, Lobotomy Corporation, 선택 중심 텍스트 어드벤처 사례
- 상태: 조건부 참고
- 주제: 캐릭터 반응과 실패 결과를 사용자의 다음 판단과 관계 기억에 연결

## 1. 질문

다음 두 문제를 동시에 해결하려면 어떻게 해야 하는가?

- 동료와 시스템이 사용자의 선택을 기억한다고 느끼게 함.
- 실패가 단순 처벌·재시작·손실이 아니라 다음 판단의 근거가 되게 함.

## 2. 관찰

### 관련성 높은 짧은 반응

선택 직후의 짧은 동료 반응은 선택을 인물 관계에 연결한다. 반응이 길거나 모든 상황에 반복되면 현재 행동과 위험 판단을 방해한다.

### 실패의 의미 기록

실패 원인, 대상 반응, 다음에 비교할 조건이 남으면 재시도는 단순 반복이 아니라 학습이 된다.

### 규칙 학습과 영구 손실은 별개

대상별 규칙과 팀 적합성을 배우는 긴장은 강하지만, 높은 영구 손실이 모든 프로젝트에 필요한 것은 아니다. 실패 학습과 자산 파괴 강도는 별도 결정이다.

### 관계 변화의 표현

사용자는 수치보다 말투, 호칭, 정보 공유, 도움 요청, 후속 장면을 통해 관계 변화를 기억하는 경우가 많다.

## 3. 일반화한 적용

```text
사용자 선택
→ 즉시 결과
→ 관련 인물의 짧은 반응
→ 관찰 가능한 실패·성공 원인
→ 선택 기억 또는 기록
→ 다음 행동·후속 장면의 차이
```

반응 분기 수준:

- 한두 문장·표정 변화.
- 짧은 추가 정보 또는 행동.
- 후속 관계 장면 해금.
- 핵심 관계에서만 장기 체인 분기.

## 4. 제외할 표면

- 특정 작품의 스킬 인격화와 문체.
- 고유 시설·관리·전투 규칙.
- 장문 내면 독백의 직접 모방.
- 영구 사망·높은 반복 손실의 자동 채택.
- 모든 선택을 거대한 분기로 확장.

## 5. 검증 질문

- 사용자가 수치보다 인물의 반응을 기억하는가?
- 실패 이유와 다음에 바꿀 행동을 설명하는가?
- 같은 실패를 정보 없이 반복하지 않는가?
- 짧은 반응이 현재 행동을 늦추거나 정답을 누설하지 않는가?
- 저장·재진입 후 선택 기억과 후속 반응이 중복되지 않는가?
- 실패 손실 강도가 학습 의욕보다 포기 의향을 더 높이지 않는가?

## 6. 재사용 가능한 원칙

- 반응의 길이보다 선택과의 관련성·타이밍이 중요하다.
- 실패는 원인과 다음 비교 근거를 남길 때 학습으로 전환된다.
- 관계 변화는 점수보다 행동과 정보 공유로 표현할 수 있다.
- 규칙 학습의 긴장과 영구 손실은 별개의 설계 선택이다.
- 대부분의 관계 선택은 작은 반응 분기로도 충분히 기억될 수 있다.

## 7. 적용 시 주의

- 수치 최적화·영구 손실이 핵심 약속인 장르에서는 더 직접적인 상태 표시가 필요할 수 있다.
- 동료가 정답을 대신 말하거나 선택하지 않은 인물이 반응하지 않게 정보 범위를 관리한다.
- 공개 사례의 실제 기능과 플레이 반응은 버전·플랫폼에 따라 달라질 수 있으므로 중요한 결정에서는 최신 원본을 다시 확인한다.
