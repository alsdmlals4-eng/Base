# 캐릭터 프로모션 포스터 모듈화 사례

- 상태: 패턴·프로젝트별 시각 검증 필요
- 출처: 사용자 제공 세로형 판타지 캐릭터 포스터와 프롬프트

## 문제

캐릭터 전신 일러스트, 이름, 특징, 인셋 표정, 세계관 카피와 타이틀을 한 장에 담으면서도 다른 캐릭터와 상세 페이지에 재사용 가능한 포스터 구조가 필요하다.

## 관찰

제공 사례는 다음 정보 슬롯을 사용한다.

1. 중앙의 3/4 전신 캐릭터.
2. 세계관을 보여주는 천상 도시 배경.
3. 상단 태그라인과 캐릭터 이름·엠블럼.
4. 우측 특징 설명 3개.
5. 좌측 세로형 카피.
6. 우측 하단 표정·측면 인셋 3개.
7. 하단 작품 타이틀, 부제, 날짜, 보조 정보.
8. 캐릭터 키 컬러와 의상·무기 모티프의 반복.

핵심은 긴 프롬프트 자체가 아니라 **캐릭터 일러스트와 정보 슬롯을 분리한 레이아웃 계약**이다.

## 채택한 구조

```text
Hero Character
+ World Background
+ Identity Header
+ Feature Modules
+ Vertical Copy
+ Portrait Insets
+ Product Title and Date
+ Footer Metadata
```

각 슬롯은 캐릭터 이름, 키 컬러, 특징 문구, 인셋 감정만 교체해 재사용한다.

## 권장 프롬프트 구조

```text
# Format
세로형 캐릭터 홍보 포스터 또는 상세 페이지 키비주얼.

# Hero
캐릭터의 대표 전신 3/4 포즈, 상징 소품, 실루엣을 가리지 않는 여백.

# Visual System
키 컬러, 보조색, 재질, 광원, 배경 세계관, 상징 모티프.

# Information Slots
상단 태그라인, 캐릭터 이름과 엠블럼, 특징 3개, 세로 카피,
표정 인셋 3개, 하단 타이틀·날짜·보조 정보.

# Protection
얼굴·손·소품의 구조를 유지하고 정보 패널이 핵심 실루엣을 가리지 않게 한다.

# Output
상업용 키비주얼 수준의 구성 시안. 최종 한글·로고는 편집 레이어로 교체한다.
```

## 텍스트 처리 결정

이미지 모델의 타이포그래피 성능이 좋아도 다음을 기본값으로 한다.

- 생성 이미지의 글자는 위치·밀도·위계 검토용 시안이다.
- 실제 이름, 설명, 날짜, 저작권은 편집 가능한 벡터·UI 레이어로 분리한다.
- 텍스트 없는 마스터 이미지를 함께 보관한다.
- 한글·영문 길이와 줄바꿈을 실제 폰트로 검수한다.

## 재사용 가능한 원칙

- 포스터의 세일즈포인트는 캐릭터와 정보 모듈이 같은 시선 흐름을 만드는 것이다.
- 설명 슬롯은 2~3개로 제한하고 각 항목은 하나의 특징만 전달한다.
- 인셋 이미지는 장식이 아니라 기본·진지·긍정 등 상태 차이를 보여준다.
- 키 컬러는 의상, 엠블럼, 구분선, 아이콘에 반복하되 본문 가독성을 해치지 않는다.
- 세로 포스터를 상세 페이지나 이벤트 배너로 바꿀 때 슬롯 우선순위만 재배치한다.

## 실패 기준

- 정보 모듈이 얼굴·무기·실루엣을 가린다.
- 모든 문구를 이미지에 굽고 수정·현지화할 수 없다.
- 인셋 표정이 메인 캐릭터와 다른 얼굴·의상으로 생성된다.
- 배경과 장식 밀도가 이름과 특징 설명보다 강하다.
- 다른 캐릭터에 적용할 때 전체 프롬프트를 처음부터 다시 작성해야 한다.

## 검증 계획

1. 같은 레이아웃에서 키 컬러와 캐릭터만 바꾼 3종을 제작한다.
2. 이름 길이와 특징 문구 길이를 바꿔 정보 슬롯의 내구성을 확인한다.
3. 텍스트 없는 마스터에 실제 한글 타이포그래피를 합성한다.
4. 모바일 상세 페이지와 세로 포스터 크롭에서 얼굴·이름·특징이 읽히는지 확인한다.

## 프로젝트 전용으로 남길 것

- 캐릭터 이름·의상·세계관·키 컬러.
- 실제 문구와 엠블럼.
- 폰트·로고·저작권 정보.
- 승인된 원본 이미지와 인셋 자산.
- 플랫폼별 최종 크기.
