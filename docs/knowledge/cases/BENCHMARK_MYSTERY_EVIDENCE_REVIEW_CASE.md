# 벤치마킹 사례 — 관찰 근거 재열람과 힌트의 경계 설계

- 출처 유형: 공개 미스터리·추리 게임 벤치마킹
- 참고 사례: The Case of the Golden Idol, Return of the Obra Dinn, PARANORMASIGHT
- 상태: 여러 프로젝트에서 조건부 참고
- 주제: 사용자가 과거 정보를 다시 검토하고 스스로 결론을 만들게 하면서 정답 누설과 메뉴 피로를 줄임

## 1. 질문

관찰·기록이 많은 콘텐츠에서 다음을 동시에 만족하려면 어떻게 해야 하는가?

- 사용자가 이미 본 근거를 쉽게 다시 찾음.
- 힌트가 정답을 직접 말하지 않음.
- 현재 장면과 기록 메뉴 사이의 왕복 피로가 낮음.
- 장소와 인물의 몰입이 기록 UI에 묻히지 않음.

## 2. 관찰

### 관찰 정보의 역할 분리

추리 만족이 높은 사례는 사실, 추정, 미확정, 결론의 역할을 구분한다. 모든 텍스트가 같은 신뢰도와 중요도로 보이지 않는다.

### 재열람 가능한 기록

사용자가 과거 장면을 기억력만으로 해결하도록 강제하지 않고, 이미 확인한 사실을 다시 볼 수 있게 한다. 재열람은 수집품이 아니라 추론 도구다.

### 힌트의 제한

힌트는 다음 행동이나 비교할 범위를 좁힐 수 있지만, 최종 연결 문장과 정답을 대신 제공하지 않는다.

### 장면과 기록의 거리

도시·인물·현재 행동은 장면에서 먼저 읽히고, 상세 기록은 현재 질문과 연결되는 경로로 연다. 기록 메뉴가 장면 자체를 대체하지 않는다.

## 3. 일반화한 적용

```text
현장 관찰
→ 출처와 상태가 있는 기록
→ 현재 질문에 관련된 기록 연결
→ 사용자의 가설
→ 검증 행동
→ 성공 또는 실패 근거
→ 기록 갱신
```

권장 정보 구분:

- 확인된 관찰.
- 해석 또는 가설.
- 도움말·탐색 방향.
- 실패·위험 사례.
- 검증된 결론.

## 4. 제외할 표면

- 특정 작품의 단어 채우기 문법.
- 흑백·파노라마·기록장 같은 고유 시각 외형.
- 동일한 사건 구성과 반전 방식.
- 메뉴 위치와 입력 구조의 직접 복제.

## 5. 검증 질문

- 사용자가 힌트 문구가 아니라 두 개 이상의 관찰 근거로 결론을 설명하는가?
- 이미 본 기록을 별도 설명 없이 다시 찾는가?
- 현재 질문과 관련 없는 기록 때문에 탐색이 길어지지 않는가?
- 실패 뒤 새 근거가 남아 같은 행동을 무작정 반복하지 않는가?
- 기록 UI를 닫은 뒤 원래 장면과 선택 상태로 자연스럽게 돌아오는가?

## 6. 재사용 가능한 원칙

- 기록은 추론을 갱신하는 도구여야 한다.
- 사실, 가설, 힌트, 실패, 결론의 신뢰 상태를 구분한다.
- 힌트는 비교 범위를 돕되 연결과 결단을 대신하지 않는다.
- 장면 몰입과 기록 재열람은 경쟁 관계가 아니라 역할 분리 문제다.
- 메뉴 외형보다 기록을 찾고 현재 질문에 연결하는 경로가 중요하다.

## 7. 적용 시 주의

- 기억 자체를 시험하는 게임은 재열람 범위를 의도적으로 제한할 수 있다.
- 모바일·패드·키보드에서 기록 왕복 비용이 다르므로 입력별 검증이 필요하다.
- 사례의 실제 제품 기능과 사용자 반응은 버전·플랫폼에 따라 달라질 수 있으므로 중요한 결정에서는 최신 원본을 다시 확인한다.
