# 사례 — 병종 도감표에서 실제 전장 스프라이트 형식으로 기준 교체

- 프로젝트: OMENWARD
- 사례 유형: 아트 형식 교체, 시각자료 상태 관리, 월드 가독성 우선
- 기록일: 2026-07-16

## 상황

한 프로젝트에 서로 다른 목적의 병종 이미지가 존재했다.

1. 10병종을 일반·엘리트·영웅·전설 열로 비교한 큰 전신 도감표.
2. 실제 게임 전장 UI 안에 소형 픽셀 유닛이 다수 배치된 플레이 화면 시안.

초기 문서화 과정에서 이미지 원본이 저장소 책임 문서와 연결되지 않아, 새 작업자가 텍스트 규칙만 보고 어떤 형식을 실제 병종 스프라이트로 제작해야 하는지 판단하기 어려운 상태가 됐다.

## 사용자 결정

병종 이미지 제작 형식은 두 번째 목적, 즉 **실제 전장에 삽입되는 소형 고해상도 픽셀 월드 스프라이트**로 변경됐다.

큰 전신 도감표는 완전히 버리지 않고 다음 용도로만 남겼다.

- 공용 병종 10개의 목록 확인.
- 같은 병종의 일반→엘리트→영웅→전설 상승 관계 비교.
- 상위 등급에서 역할 실루엣과 무기가 강화되는 방향 탐색.

다음 요소는 더 이상 실제 스프라이트 기준이 아니게 됐다.

- 큰 전신 캐릭터 비율.
- 도감용 렌더링 밀도.
- 등급별 정확한 의상·장식과 포즈.
- 축소하지 않은 캐릭터 일러스트 중심 제작 방식.

## 적용한 문서 구조

```text
시각자료 인덱스
→ 최신 이미지를 APPROVED_DIRECTION_REFERENCE로 지정
→ 도감표를 SUPERSEDED_FOR_SPRITE_FORMAT / PARTIAL_REFERENCE로 강등
→ 병종 비주얼 형식 책임 문서 생성
→ Documentation Map과 인수인계 문서에 라우팅
```

새 책임 문서에는 다음을 명확히 했다.

- 인간형 약 2.5~3등신 전술 미니어처 비율.
- 34~40px 첫 월드 표시 높이 가설.
- 얼굴보다 무기·자세·몸통 덩어리와 공격 방향 우선.
- 아군·적군은 공용 AnimationContract를 공유하고 Visual Set만 다름.
- 등급 위계는 단순 크기 확대보다 기능적인 실루엣·무기·자세·제한적 VFX로 표현.
- 도감·초상화와 월드 스프라이트는 같은 디자인 언어를 공유하되 별도 자산으로 제작.

## 이미지 전체를 승인하지 않은 이유

실제 전장 시안에는 임시 수치, 문구, 맵 구조, 건물 위치와 미니맵처럼 보이는 UI도 함께 있었다. 유닛 형식이 승인됐다고 화면 전체가 승인된 것은 아니다.

따라서 다음을 별도로 기록했다.

```text
참고할 것
- 유닛 크기와 픽셀 밀도
- 역할 실루엣
- 진영 구분
- 전장과 HUD의 정보 계층

참고하지 않을 것
- 임시 텍스트와 수치
- 이미지의 맵 토폴로지
- 건물 배치와 비용
- 미니맵 형태
```

## 얻은 교훈

### 1. 용도가 다른 이미지는 경쟁 기준이 아니다

도감표는 비교와 캐릭터 탐색에 강하고, 플레이 화면 시안은 실제 크기·중첩·배경 대비 검증에 강하다. 하나를 채택했다고 다른 자료의 모든 가치가 사라지는 것은 아니다.

### 2. 최신 기준은 파일명보다 상태가 책임진다

`final`, `new`, `v2` 같은 파일명보다 시각자료 인덱스의 승인·부분 참고·교체 상태가 안전하다.

### 3. “이 이미지처럼”을 요소 단위로 해석해야 한다

사용자는 유닛 형식을 지칭했지만 이미지에는 UI·맵·수치도 포함돼 있었다. 어떤 층을 채택했는지 명시하지 않으면 시스템까지 잘못 복제할 수 있다.

### 4. 실제 사용 환경에서 먼저 검증한다

월드 스프라이트는 전신 원화만 보고 승인하지 않는다. 1080p·720p 전장에 여러 기를 배치하고 병종, 진영, 공격 방향과 등급을 검증한다.

### 5. 누락된 원본과 문서화 완료를 구분한다

이미지의 존재를 확인하고 인덱스에 등록했어도 저장소 바이너리 이동이 끝나지 않았다면 `MIGRATION_PENDING`이다. 이 구분이 인수인계 신뢰성을 지킨다.

## 다른 프로젝트에 적용하지 말아야 할 경우

- 유닛이 항상 큰 카드나 초상화로만 등장하고 월드 전투 표현이 없는 게임.
- 진영마다 전투 프레임과 체형이 완전히 달라 공용 애니메이션 계약이 제품 목표가 아닌 게임.
- 등급 상승이 외형 변화 없이 UI 배지로만 표현되는 것이 의도인 게임.
- 실시간 다수 전투보다 단일 캐릭터 감상과 표정 연기가 핵심인 게임.

이 사례의 핵심은 특정 픽셀 비율을 복제하는 것이 아니라, **이미지의 목적을 분리하고 최신 형식 결정과 잔존 참고 가치를 동시에 문서화하는 방법**이다.
