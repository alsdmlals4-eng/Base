# 사례 — 특수 이동 경로를 필요할 때만 공개하기

- 출처 프로젝트: OMENWARD
- 상태: 채택 / PoC 검증 전
- 주제: 전문 병종의 우회 경로를 평상시 숨기고 선택·배치 중에만 표시

## 1. 문제

전문 침투 병종이 사용하는 우회 경로를 전장에 항상 표시하면 플레이어는 기능을 쉽게 이해할 수 있지만 다음 문제가 발생한다.

- 주 도로와 목표 지점의 시각 우선순위를 약화.
- 사용하지 않는 동안에도 화면 복잡도가 증가.
- 비밀 침투와 긴장감이 약해짐.
- 일반 유닛도 사용할 수 있는 길처럼 오해할 수 있음.
- 위험 범위·점령 표시와 선이 겹침.

반대로 경로를 완전히 숨기면 선택 결과와 도착 위치를 예측하기 어렵다.

## 2. 채택한 결정

```text
평상시
→ 경로·입구·도착점 숨김

전문 병종 선택
→ 사용 가능한 경로 윤곽 표시

라인 지정
→ 출발점·방향·예상 도착 구역 표시

배치 확정
→ 필요한 동안만 진행 상태 표시

침투 종료 또는 선택 해제
→ 다시 숨김
```

경로는 안개·연무로 가리고, 위험 공격 전조와 다른 색·형태 언어를 사용한다.

## 3. 정보 공개 원칙

플레이어에게 보여준다:

- 어떤 라인으로 들어가는지.
- 대략 어느 후방 구역에 도착하는지.
- 배치 확정 전 취소 가능 여부.
- 도착까지 남은 시간 또는 진행 상태.

상대에게 보여준다:

- 대응 시간이 필요한 경우 도착 직전 경고 구역.
- 정확한 출현 좌표가 아니라 대응 가능한 범위.

평상시에는 보여주지 않는다:

- 사용하지 않는 라인의 전체 경로.
- 일반 유닛이 쓸 수 있다고 오해할 출입구.
- 전투 위험 범위처럼 보이는 강한 선과 색.

## 4. 장점

- 전문 병종 선택 순간에만 관련 정보가 나타나 학습과 조작이 연결됨.
- 평상시 전장 가독성 유지.
- 침투 역할의 비밀성과 긴장감 유지.
- 도착 구역을 예고해 불공정한 즉사·기습을 완화.

## 5. 위험

- 숨겨진 기능을 플레이어가 발견하지 못할 수 있음.
- 경로 표시가 짧거나 약하면 배치 결과를 이해하지 못함.
- 상대 경고가 너무 늦으면 대응 불가, 너무 빠르면 전문 병종 가치가 사라짐.
- 연무가 공격 전조나 시야 제한 효과와 혼동될 수 있음.

## 6. 보완 설계

- 첫 사용에서는 안내자·툴팁·배치 미리보기로 기능을 명확히 소개.
- 선택 가능한 병종 카드에 우회 태그와 도착 아이콘 표시.
- 경로 선택 중 다른 전투 UI보다 위에 표시하되 핵심 목표 UI는 가리지 않음.
- 상대 경고는 정확한 개체 위치보다 도착 구역과 남은 시간을 전달.
- 경로 표시는 선택 취소 시 즉시 사라짐.

## 7. 검증 방법

- 첫 사용자에게 설명 없이 전문 병종을 선택하게 하고 배치 가능한 라인과 도착 위치를 예측하는지 관찰.
- 경로 표시 전·후 전장 상태 탐색 시간을 비교.
- 상대가 경고를 보고 의미 있는 대응을 할 수 있는지 측정.
- 공격 범위 전조와 우회 경로를 혼동하는 비율 확인.
- 일반 유닛으로 경로를 클릭하거나 사용할 수 있다고 오해하는지 확인.

## 8. 재사용 가능한 원칙

- 항상 필요하지 않은 공간 정보는 관련 행동이 시작될 때 공개한다.
- 숨김은 정보 삭제가 아니라 맥락 기반 공개 설계다.
- 전문 기능은 선택, 미리보기, 확정, 실행, 종료의 각 단계에서 다른 정보를 요구한다.
- 기습 기능도 상대에게 최소한의 대응 신호를 제공해야 공정성이 유지된다.
- 같은 화면의 선·색·연무는 서로 다른 의미를 명확히 구분해야 한다.

## 9. 적용 가능한 다른 사례

- 비밀 통로·잠입 경로.
- 건설 가능 영역.
- 사거리와 영향 범위.
- 특정 장비를 들었을 때만 보이는 탐색 단서.
- 비행·수중·지하 전용 이동 레이어.

모든 경우에 평상시 가독성, 선택 중 예측 가능성, 실행 후 피드백을 함께 검증한다.
