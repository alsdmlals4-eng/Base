# 기존 게임 프로젝트 안전 마이그레이션 방법

- 상태: 공용 원칙·호환 경로
- 실행 Skill: `skills/managing-game-project-operating-system/SKILL.md`
- 실행 mode: `audit` → 사용자 승인 → `migrate` → `verify`

이 문서는 기존 참조를 보존하면서 안전 마이그레이션의 불변 원칙만 책임진다. 상세 파일 인벤토리, 처리표, 이동·삭제와 Health Review는 통합 Skill이 실행한다.

## 최우선 원칙

```text
기존 내용 보존
→ 현재 책임과 참조 관계 파악
→ 중복·충돌·누락 분석
→ 목표 구조·보존·롤백 제안
→ 사용자 검토
→ 승인된 처리표만 변경
→ 변경 전후 보존·참조·발행 검증
```

Base 구조를 그대로 복제하거나 프로젝트를 강제로 개명하지 않는다.

## 절대 보존 대상

조사와 사용자 승인 없이 다음을 삭제·축약·변경하지 않는다.

- 사용자 승인 기획과 결정
- 세계관·용어·수치·예외
- 현재 구현 상태와 실제 경로
- 승인 이미지·UI·다이어그램·프롬프트
- 테스트 결과·실패 사례·미검증 기록
- Roadmap·Issue·Plan·PR 진행 이력
- Active Context의 미완료 작업
- `[보류]`, `[확인 필요]`, `[미검증]`
- 외부에서 참조되는 파일과 승인·감사 근거

## 권한 단계

### Audit only

현재 구조·책임 지도·참조·고유 정보·중복·충돌·누락·위험·목표 처리표만 작성한다. 명백한 오탈자와 복구 가능한 작은 링크 수정 외에는 구조를 변경하지 않는다.

### Approved migration

사용자가 승인한 처리표의 항목만 수행한다. 불확실성은 `[확인 필요]`로 유지한다.

### Cleanup·enforcement

보존 대조와 참조·발행·콜드 스타트 검증이 끝난 후 완전히 흡수된 중복만 제거한다. GitHub Actions와 브랜치 보호는 실제 활성 상태를 별도로 검증한다.

## 파일 인벤토리

| 현재 경로 | 역할 | 참조 | 고유 정보 | 중복·충돌 | 상태 | 제안 | 위험 | 검증 |
|---|---|---|---|---|---|---|---|---|

한 파일에 여러 책임이 섞여 있으면 요소 단위로 기록한다. 고유 문장·표·수치·이미지·예외·결정 이유·미검증 항목이 하나라도 있으면 완전 중복으로 판정하지 않는다.

## 수명주기

- `[현행]`: 실제로 참조하는 책임 원본·실행 Skill
- `[백업]`: Git 이력만으로 충분하지 않은 외부 원본·감사·승인 근거
- `[보류]`: 재개 조건과 선행 작업이 있는 미래 항목
- `[제거 후보]`: 완전히 흡수됐을 가능성이 있으나 삭제 승인이 없는 항목

단순 이전 버전은 별도 복제본 대신 Git 이력으로 보존한다.

## 삭제 조건

다음을 모두 충족해야 한다.

- 모든 고유 정보·표·이미지·예외·보류가 현행 원본에 승계됨
- 활성·보조·외부 참조가 새 경로로 갱신됨
- 발행본·Manifest가 검증됨
- Git 이력 또는 별도 보존으로 복구 가능함
- 변경 전후 보존 대조가 통과함
- 사용자 승인 또는 승인된 처리표가 있음

확신할 수 없으면 삭제하지 않고 `[제거 후보]`로 유지한다.
