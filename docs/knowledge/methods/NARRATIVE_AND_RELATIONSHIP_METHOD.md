# 서사·대화·관계 설계 방법

- 상태: 공용 방법
- 목적: 장면·대사·선택·관계를 설정 설명과 분량 채우기가 아니라 사용자 행동과 이후 기억에 연결한다.

## 1. 장면의 최소 단위

한 장면은 다음 질문에 답해야 한다.

1. 왜 지금 필요한가?
2. 사용자가 무엇을 새로 보거나 선택하는가?
3. 장면 전후에 무엇이 달라지는가?
4. 어떤 인물·세계·규칙을 행동으로 보여주는가?
5. 다음 장면과 무엇으로 연결되는가?

권장 구조:

```text
상황과 변화
→ 인물의 행동·대화
→ 사용자 선택·해석
→ 즉시 반응
→ 상태·기억·기록
→ 다음 질문 또는 여운
```

한 장면에 여러 핵심 정보와 감정 전환을 동시에 넣지 않는다.

## 2. 대화의 세 모드

### 기능 대화

- 현재 행동, 위험, 비교 기준을 전달한다.
- 짧고 직접적이다.
- 미확인 사실을 단정하지 않는다.
- 현재 선택을 대신하지 않는다.
- 최초 상세와 재방문 축약을 분리한다.

### 관계 대화

- 인물 간 거리·가치관·기억 변화를 보여준다.
- 말투, 침묵, 물건, 시선, 행동 순서를 사용한다.
- 선택의 차이는 다음 반응과 정보 공유에 남긴다.
- 성능 보상보다 장면과 태도 변화가 우선한다.

### 세계관 대화

- 조직·문화·역사·규칙을 생활과 갈등 속에서 보여준다.
- 백과사전식 설명을 피한다.
- 두 인물이 같은 사실을 다르게 해석하게 한다.
- 장면 목적과 무관한 설정은 기록 문서나 별도 콘텐츠로 보낸다.

한 장면에서 한 모드를 중심으로 두고 다른 모드는 보조한다.

## 3. 상황지시문

감정을 평가하기보다 관측 가능한 차이를 제시한다.

```text
공간·소리·빛·사물의 변화
→ 인물·장비의 구체적 반응
→ 아직 확인되지 않은 요소
→ 사용자가 고를 수 있는 행동 여지
```

나쁜 예:

> 방은 매우 무섭고 불길했다.

좋은 예:

> 꺼져 있던 복도등이 문 안쪽부터 한 칸씩 켜졌다. 마지막 전등은 켜지지 않았지만, 그 아래의 그림자는 먼저 움직였다.

## 4. 캐릭터 목소리 카드

캐릭터마다 다음을 기록한다.

- 핵심 욕망.
- 두려움과 회피 행동.
- 책임을 느끼는 대상.
- 판단 기준.
- 말의 길이와 속도.
- 자주 쓰는 문장 구조.
- 감정을 숨기는 방식.
- 갈등 시 바뀌는 말투.
- 친밀해졌을 때만 보이는 행동.
- 절대 하지 않을 말과 행동.

목소리는 어휘 목록보다 **어떤 상황에서 무엇을 먼저 보는가**로 구분한다.

## 5. 설명보다 행동

설정을 직접 설명해야 할 때도 행동과 선택을 먼저 둔다.

```text
규정 위반 행동
→ 다른 인물의 제지
→ 책임 기준 충돌
→ 필요한 규칙 한 문장
```

세계관 정보는 다음 중 하나와 연결될 때 장면에 넣는다.

- 현재 선택의 이유.
- 인물 갈등.
- 다음 사건의 준비.
- 사용자의 새로운 질문.
- 기록·도감·메뉴의 해금.

## 6. 선택지 설계

선택지는 정답 문장과 오답 문장을 감추는 장치가 아니다. 서로 다른 태도·위험·정보·비용을 가져야 한다.

선택지 유형:

- 행동 방식.
- 정보 우선순위.
- 사람과 절차의 우선순위.
- 의심 유지 또는 신뢰.
- 책임 수용 또는 분담.
- 즉시 해결 또는 기록 보존.

좋은 선택지는 선택 직후 행동의 방향을 예측할 수 있다.

## 7. 관계 설계

### 숫자와 경험 분리

내부 계산이 필요하더라도 사용자는 다음으로 관계 변화를 느낀다.

- 말투와 호칭.
- 먼저 말하거나 기다리는 순서.
- 공유하는 정보의 범위.
- 도움 요청과 거절 방식.
- 장면의 거리와 시선.
- 후속 이벤트 해금.
- 기록 문장의 변화.

관계는 성능 보너스보다 인물 이해와 장면 차이를 우선한다.

### 태그 예시

- 신뢰.
- 경계.
- 부채.
- 경쟁.
- 공감.
- 보호.
- 균열.

프로젝트의 핵심 관계 질문에 맞게 선택한다.

### 선택 기억

```text
선택 ID
→ 대상 인물
→ 기억할 태도
→ 발동 가능한 후속 반응
→ 충돌 시 우선순위
→ 소비 또는 유지 여부
```

같은 선택을 반복해 관계 변화와 보상을 중복 획득하지 않게 한다.

## 8. 분기 예산

| 분기 수준 | 변화 | 사용 조건 |
|---|---|---|
| 반응 분기 | 한두 문장·표정·행동 차이 | 대부분의 태도 선택 |
| 장면 분기 | 짧은 추가 장면·정보 공유 | 기억할 가치가 있는 관계 선택 |
| 체인 분기 | 다음 에피소드의 진입·순서 변화 | 핵심 관계 축 |
| 결말 분기 | 장기 결과와 엔딩 변화 | 프로젝트 핵심 약속일 때만 |

많은 작은 반응이 적은 큰 분기보다 ‘선택을 기억한다’는 감각을 줄 수 있다.

## 9. 연속 이벤트

```text
갈등 또는 질문 제시
→ 서로의 방식이 실제 문제를 만남
→ 선택의 비용이나 약점이 드러남
→ 상대 방식을 일부 이해함
→ 새로운 공동 행동 또는 남은 균열
```

같은 갈등을 말만 바꿔 반복하지 않는다.

## 10. 실패와 후일담

후속 장면은 처벌 설명보다 의미 정리를 담당한다.

- 무엇을 잘못 보았는가.
- 누가 어떤 비용을 감당했는가.
- 다음에 무엇을 다르게 할 것인가.
- 관계에 어떤 흔적이 남았는가.
- 기록에 무엇을 남기거나 숨길 것인가.

후일담은 정답을 다시 읽는 장면이 아니라 책임과 기억의 해석을 보여준다.

## 11. 데이터 경계

- 대사 본문은 결과를 새로 계산하지 않는다.
- 선택 결과는 한 곳에서 정의한다.
- 관계 상태와 장면 연출 상태를 분리한다.
- 표정·컷인 파일 경로를 대사 문장에 직접 하드코딩하지 않는다.
- 기록·도감은 이미 발생한 사실을 요약하며 새 필수 사실을 만들지 않는다.
- 자산 부재와 연출 비활성 상태에서도 장면이 진행돼야 한다.

## 12. 작성 절차

1. 장면의 역할과 새 정보 하나를 정한다.
2. 등장인물별 현재 욕망과 판단 기준을 적는다.
3. 대화 없이 행동만으로 장면 흐름을 작성한다.
4. 필요한 대사만 추가한다.
5. 사용자 선택과 즉시 반응을 넣는다.
6. 상태·기억·기록 변화를 정의한다.
7. 다음 장면과의 연결을 적는다.
8. 말투·정보 누설·분량·반복을 검수한다.

## 13. 사례 — 수치를 말하지 않고 같은 사실을 더 구체적으로 말하기

상대의 위험도·성장 상태·전법처럼 내부에는 정확한 값이 있지만, 기본 화면과 대사에서 숫자표를 직접 보여주면 인물이 개발용 태그처럼 보일 수 있다. 반대로 분위기만 있는 문구를 쓰면 사용자는 선택 근거를 얻지 못한다.

해결은 같은 내부 의미 키에 대해 정보 단계별 문장 역할을 정하는 것이다.

```text
내부 의미 키
→ 최초 정보: 방향과 위험 범주만 암시
→ 반복 정보: 같은 어휘 계열로 의미를 학습
→ 상세·정탐 정보: 같은 사실을 더 구체적으로 표현
→ 결과 장면: 실제 행동으로 문구의 의미를 입증
```

예시 형식:

- 최초: “한 계통만을 오래 벼렸다는 소문이 있다.”
- 반복: “다른 기술보다 같은 초식을 거듭 다듬었다고 한다.”
- 상세: “상위 기술의 문턱에 가까우며 보조 계통의 흔적은 적다.”

작성 규칙:

- 문구는 선택 근거를 제공하지만 정답 행동을 대신 말하지 않는다.
- 높은 정보 단계는 이전 문장을 뒤집지 않고 구체성만 높인다.
- 같은 의미에 같은 어휘 계열을 사용한다.
- 강한 상태를 약한 표현으로 속이지 않는다.
- 기능 대사는 현재 위험과 비교 기준을 짧게 말하고, 세계관 대사는 왜 그런 평가가 생겼는지 생활·전적·관계 속에서 보여준다.
- 현지화에서도 강도 단계와 집중·균형·변칙 같은 성향 차이가 유지되는지 검수한다.

이 패턴은 적 정보뿐 아니라 관계 상태, 조직 평판, 감염 단계, 신뢰도와 조사 진행에도 적용할 수 있다. 정밀 수치 비교가 핵심인 분석 화면에는 세계관 문구만 강제하지 않고 상세 숫자 보기나 개발·접근성 경로를 함께 제공한다.

## 14. 완료 기준

- [ ] 장면의 중심 질문이 하나다.
- [ ] 대사를 가려도 기본 행동 흐름을 이해한다.
- [ ] 말투만으로 주요 인물을 구분한다.
- [ ] 설정은 행동·갈등·선택에 연결돼 있다.
- [ ] 선택지가 서로 다른 태도·위험·정보를 가진다.
- [ ] 관계 변화가 숫자보다 반응으로 보인다.
- [ ] 실패와 후일담이 다음 판단이나 감정을 남긴다.
- [ ] 미확인 정보와 정답을 안내자가 누설하지 않는다.
- [ ] 반복 재생·저장 복구에서 보상과 기억이 중복되지 않는다.
- [ ] 피해와 후유증을 가벼운 농담으로 소비하지 않는다.
