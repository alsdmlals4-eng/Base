# 캐릭터·서사형 콘텐츠 아트 디렉션 방법

- 상태: 공용 방법
- 목적: 초상·전신·대화 장면·관계 이벤트 중심 프로젝트에서 캐릭터의 정체성과 화면 가독성을 반복 제작 가능한 규격으로 만든다.

기존 `ART_DIRECTION_METHOD.md`가 전장·유닛·구조물의 실제 플레이 크기와 기능 구분을 넓게 다룬다면, 이 문서는 캐릭터와 서사 장면의 얼굴·자세·감정·텍스트 없는 자산 운영에 집중한다.

## 1. 시각 약속

좋은 시각 약속은 다음을 포함한다.

```text
세계·대상
+ 핵심 분위기
+ 사용자가 먼저 봐야 할 인물·정보
+ 차별화되는 형태·색·재료
```

예시 형식:

> 익숙한 도시 공간을 절제된 기록물 질감과 제한된 신호색으로 표현해, 인물의 표정과 비정상적인 변화가 먼저 읽히게 한다.

## 2. 스타일 혼합표

여러 스타일을 섞을 때 역할을 분리한다.

| 요소 | 역할 | 사용 범위 | 과잉 시 문제 |
|---|---|---|---|
| 기본 렌더링 | 전체 일관성 | 대부분의 인물·배경 | 화면 전체가 같은 밀도 |
| 질감 | 장르·재료감 | 기록, 오염, 특정 공간 | 얼굴·텍스트 가독성 저하 |
| 강조 스타일 | 능력·감정·브랜드 순간 | 컷인, 핵심 장면 | 특별함 소실 |
| 실험 스타일 | 회상·특수 상태 | 제한된 장면 | 기본 화면과 규칙 없는 충돌 |

정확한 비율은 프로젝트에서 정하되, 각 스타일이 어떤 장면 역할을 갖는지 설명한다.

## 3. 캐릭터 디자인 카드

- 역할과 첫인상.
- 핵심 욕망과 태도.
- 실루엣 키워드.
- 대표 자세와 무게 중심.
- 얼굴·헤어·의상에서 가장 중요한 식별 요소.
- 고유 강조색.
- 업무·전투·일상 소품.
- 감정을 숨기는 방식.
- 자주 필요한 표정.
- 다른 캐릭터와 겹치면 안 되는 요소.
- 자산 티어와 제작 우선순위.

외형과 서사 역할을 연결한다. 방어적인 인물은 반드시 두꺼운 갑옷을 입을 필요가 없지만, 다른 인물 앞에 서거나 공간을 막는 자세로 역할을 보여줄 수 있다.

## 4. 공통 시각 축

### 형태

- 성인·청소년 등 인체 비율 기준.
- 선화 굵기와 색.
- 실루엣의 각·곡선·비대칭.
- 손·소품·시선이 읽히는 크기.

### 명암과 재료

- 셰이딩 단계.
- 피부·천·금속·유리·종이의 광택 상한.
- 광원 방향과 배경 대비.
- 얼굴과 손의 정보 우선순위.

### 색

- 기본 중성색.
- 브랜드 신호색.
- 캐릭터 식별색.
- 위험·성공·상호작용 색과의 충돌 방지.
- 색상 없이도 구분할 형태·문양.

### 질감과 후처리

- 종이, 잉크, 필름, 노이즈 등.
- 기본 화면과 회상·오염 장면의 강도 차이.
- 텍스트와 얼굴을 해치지 않는 상한.

## 5. 자산 계층

1. 전신 — 편성·소개·프로필.
2. 반신 초상 — 대화·일상·후일담.
3. 표정 파생 — 감정과 관계 변화.
4. 행동 컷 — 조사·지원·작업.
5. 핵심 장면 컷 — 결정적 관계·규칙·결말.
6. 특수 상태 — 오염·변형·회상.

모든 자산을 처음부터 만들지 않는다. 실제 장면 빈도와 감정 중요도에 따라 T0~T3로 우선순위를 둔다.

## 6. 표정 자산

표정은 파일 번호보다 의미 키로 관리한다.

- `neutral`
- `focus`
- `alert`
- `anger`
- `concern`
- `soft`
- `awkward`
- `hurt`
- `resolve`
- `special_state`

모든 인물이 모든 키를 가질 필요는 없다. 공통 의미를 캐릭터 고유 강도와 자세로 표현한다.

제작 우선순위:

1. 기본.
2. 집중.
3. 경계.
4. 캐릭터 핵심 감정 2종.
5. 관계 변화용 표정.
6. 특수 상태.

자산이 없으면 가까운 감정군 또는 기본 표정으로 폴백한다.

## 7. 레퍼런스 보드

작품 이름 목록이 아니라 축별 관찰 자료로 만든다.

- 형태·실루엣.
- 색·조명.
- 재료·질감.
- 얼굴·손·포즈.
- 레이아웃과 정보 위계.
- 피해야 할 반례.

각 이미지에 관찰 요소, 적용 원리, 복제 금지 요소, 제작 제약을 적는다.

## 8. 생성형 이미지 사용

- 이미지 안에 이름·대사·버튼·상태 텍스트를 굽지 않는다.
- 캐릭터 정체성은 프롬프트만이 아니라 별도 디자인 카드로 관리한다.
- 레퍼런스 이미지가 정체성·포즈·재료 중 무엇을 담당하는지 기록한다.
- 원본 출력과 후처리 파일을 분리한다.
- 알파, 크기, 얼굴 위치, 여백, 텍스트 없음 여부를 manifest에 기록한다.
- 작가명보다 형태·광원·재료·구도·사용 목적을 설명한다.
- 생성 결과의 문자·로고·복잡한 문양은 신뢰하지 않고 UI·벡터로 분리한다.
- 개별 결과가 흔들리면 반복 재생성보다 공통 레퍼런스·프롬프트 구조를 고친다.

## 9. 실제 화면 검증

- 대화창·선택지·HUD와 함께 배치한다.
- 실제 크롭과 최소 해상도에서 얼굴·손·핵심 소품을 확인한다.
- 두 명 이상 배치했을 때 실루엣과 강조색이 겹치지 않는지 본다.
- 표정 차이가 축소 화면에서 읽히는지 확인한다.
- 배경과 인물의 명암이 현재 화자를 구분하는지 본다.
- 자산 누락 시 기본 이미지·실루엣으로 기능이 유지되는지 확인한다.

## 10. 실패 기준

- 캐릭터마다 다른 렌더링 규칙을 사용함.
- 화려한 원화만 보고 실제 화면 크롭을 검증하지 않음.
- 생성 이미지 안의 글자를 그대로 사용함.
- 모든 표정을 먼저 만들고 장면 우선순위를 나중에 정함.
- 색 외에는 캐릭터를 구분할 수 없음.
- 배경·VFX가 얼굴과 선택을 가림.
- 자산 하나가 없으면 장면 진행이 중단됨.

## 11. 완료 기준

- 축소 화면에서도 인물과 핵심 감정이 읽힌다.
- 캐릭터가 서로 구분되지만 같은 작품에 속해 보인다.
- 제작자가 추가 질문 없이 기본·표정·행동 변형을 만들 수 있다.
- 이미지와 의미 텍스트가 분리돼 수정·현지화 가능하다.
- 실제 화면과 접근성 옵션에서 가독성을 검증한다.
- 자산 우선순위와 폴백이 명시돼 있다.
