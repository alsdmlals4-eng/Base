# 시각 참고자료 유입·교체·인수인계 방법

- 유형: 공용 Method
- 적용 대상: 게임 기획, 아트 디렉션, UI/UX, 레벨 디자인, 연출, 벤치마킹
- 목적: 대화·로컬·파일 라이브러리에 남은 이미지를 프로젝트 문서에서 누락하거나 오래된 시안을 최신 승인안으로 오인하는 일을 방지한다.

## 1. 핵심 원칙

이미지 파일을 저장하는 것만으로는 인수인계가 끝나지 않는다.

```text
이미지 원본
+ 상태
+ 참고할 요소
+ 참고하지 않을 요소
+ 현재 결정과 달라진 요소
+ 연결되는 책임 문서
```

이 여섯 항목이 함께 있어야 다른 작업자와 AI가 안전하게 재사용할 수 있다.

## 2. 권장 상태

```text
APPROVED_DIRECTION_REFERENCE
PARTIAL_REFERENCE
EXPLORATION
SUPERSEDED
ARCHIVE_SOURCE
MIGRATION_PENDING
```

### APPROVED_DIRECTION_REFERENCE

최신 사용자 결정이나 승인 문서가 특정 방향을 공식 기준으로 채택한 상태다. 이미지 전체가 아니라 명시된 요소만 권위가 있다.

### PARTIAL_REFERENCE

일부 요소만 참고한다. 사용할 것과 사용하지 않을 것을 반드시 분리한다.

### EXPLORATION

아이디어 탐색 시안이다. 승인 사양이나 제작 규격으로 사용하지 않는다.

### SUPERSEDED

더 최신 결정으로 교체됐다. 삭제보다 교체 이유와 남겨둘 가치가 중요하다.

### ARCHIVE_SOURCE

과거 결정·변경 이력·레거시 문서의 출처다. 활성 작업의 시작점으로 사용하지 않는다.

### MIGRATION_PENDING

존재는 확인했으나 저장소 또는 공식 자산 경로로 실제 이동되지 않았다. 이 상태에서 이전 완료라고 보고하지 않는다.

## 3. 이미지 한 장의 기록 형식

```markdown
### VR-000 — 이름

- 상태:
- 원본 출처와 날짜:
- 저장 경로:
- 연결 책임 문서:

참고할 것:
- ...

참고하지 않을 것:
- ...

현재 결정과 달라진 것:
- ...
```

파일명은 내용과 상태를 추측할 수 있게 작성하되, 승인 상태는 파일명보다 인덱스가 책임진다.

## 4. 유입 절차

사용자가 이미지·영상 프레임·도표·스크린샷을 제공하면 작업 종료 전에 다음을 수행한다.

1. 사용 가능한 원본 또는 무손실·고품질 변환본을 프로젝트 자산 경로에 저장한다.
2. 시각자료 인덱스에 ID, 날짜, 상태, 출처와 경로를 등록한다.
3. 참고할 것, 참고하지 않을 것, 현재 결정과 다른 것을 기록한다.
4. 관련 GDD·아트·UI·연출·레벨 디자인 책임 문서에서 인덱스를 읽도록 연결한다.
5. Codex나 외부 작업자 Work Order의 읽기 순서에 포함한다.
6. 기존 기준과 교체 관계가 있으면 이전 자료를 `SUPERSEDED`로 낮추고 교체 이유를 기록한다.
7. 실제 파일 이동을 확인하지 못했으면 `MIGRATION_PENDING`을 유지한다.

## 5. 이미지에서 기획 사양을 추출하는 방법

이미지는 다음 네 층으로 분해한다.

### A. 형식

- 화면 종횡비와 카메라 거리.
- 스프라이트 또는 일러스트의 비율.
- 픽셀·벡터·셀·페인터리 등 렌더링 방식.
- UI 패널과 월드 오브젝트의 결합 방식.

### B. 정보 계층

- 가장 먼저 보이는 요소.
- HUD, 전장, 캐릭터, 효과의 시선 순서.
- 작은 크기에서 역할을 전달하는 단서.
- 색상 없이도 남는 실루엣 정보.

### C. 표면 요소

- 팔레트, 재질, 외곽선, 장식, 조명.
- 임시 텍스트, 수치, 아이콘과 생성형 이미지 오류.

### D. 시스템 암시

- 이미지에 존재하는 것처럼 보이는 맵, 건물, 미니맵, 스킬, 자원과 상호작용.

D층은 가장 위험하다. 시안에 보인다는 이유만으로 실제 시스템 사양으로 채택하지 않는다. 시스템 책임 문서와 충돌하면 책임 문서가 우선한다.

## 6. 교체 결정 기록

사용자가 “두 번째가 아니라 첫 번째 형식으로 바꾼다”처럼 기준을 교체하면 다음을 기록한다.

```text
새 기준의 정확한 요소
이전 자료에서 계속 참고할 요소
더 이상 참고하지 않을 요소
영향받는 제작 규격
영향받지 않는 시스템 규칙
```

예를 들어 큰 캐릭터 도감표에서 실제 전장 월드 스프라이트로 형식을 바꾸더라도, 도감표의 병종 목록과 등급 비교 구조는 부분 참고로 남길 수 있다. 반대로 캐릭터 비율과 렌더링 밀도는 폐기될 수 있다.

## 7. 누락 감사

새 인수인계 또는 문서 정리 시 다음 위치를 함께 조사한다.

- 현재 대화 첨부.
- 프로젝트 저장소의 images, art, references, docs, archive.
- 사용자의 File Library 또는 공유 드라이브.
- 이전 GDD·PDF·PPTX·DOCX의 삽입 이미지.
- HTML 맵 툴, 보드, 내보낸 PNG·JSON.
- Issue·PR 댓글 첨부.
- 로컬 산출물과 zip.

찾은 자료는 곧바로 활성 기준으로 올리지 않는다. 먼저 관련성과 최신성을 판단하고 상태를 부여한다.

## 8. Codex Work Order 규칙

시각 작업 Work Order에는 다음을 포함한다.

- 시각자료 인덱스 경로.
- 최신 승인 기준 이미지 ID.
- 부분 참고와 폐기 자료 ID.
- 이미지의 비권위 요소.
- 실제 화면 크기와 검증 해상도.
- 산출물 형식과 비교 캡처 요구.
- 사용자 승인 전 자산 대량 제작 금지.

Codex는 이미지 파일만 보고 기획을 추론하지 않고, 인덱스와 책임 문서를 함께 읽는다.

## 9. 완료 체크리스트

- [ ] 이미지 파일이 공식 경로에 존재한다.
- [ ] 시각자료 인덱스에 등록됐다.
- [ ] 승인·부분 참고·탐색·폐기 상태가 명확하다.
- [ ] 참고할 것과 참고하지 않을 것이 구분됐다.
- [ ] 시스템 책임 문서와 충돌 여부를 확인했다.
- [ ] 교체된 자료의 `SUPERSEDED` 이유가 남아 있다.
- [ ] 관련 Work Order와 Documentation Map에 연결됐다.
- [ ] 이동하지 못한 자료를 완료로 보고하지 않았다.

## 10. 피해야 할 실패

- 이미지가 대화에 있으니 다음 작업자도 볼 수 있다고 가정한다.
- 파일만 저장하고 어떤 부분이 승인됐는지 기록하지 않는다.
- 오래된 도감·UI 시안을 최신 제작 형식으로 사용한다.
- 이미지 안의 임시 수치와 시스템을 GDD보다 우선한다.
- 새 기준을 등록하면서 이전 자료를 이유 없이 삭제한다.
- 파일 라이브러리에서 발견했다는 사실을 저장소 이동 완료로 표현한다.
- 외부 게임 스크린샷을 그대로 제작 자산이나 레이아웃 원본으로 복제한다.
