# 애니메이션·전투 연출 실무 스킬 매트릭스

- 상태: 공용 스킬 계약
- 목적: 이동·공격·피격·사망·상호작용·승리 연출을 전투 데이터와 정확히 연결하고 다수 객체 환경에서 읽히게 한다.

## 1. 상태 설계

필요 능력:

- 유닛 역할별 필수 상태를 식별.
- 공통 상태와 특수 상태를 구분.
- 상태 전이와 취소 조건 정의.
- 코드·데이터·자산의 상태 ID를 통일.

산출물:

- 상태 목록.
- 전이도.
- 취소·우선순위 표.
- 누락 자산 검증 규칙.

실패:

- 애니메이션 이름은 있지만 런타임 상태와 연결되지 않음.
- 특수 행동을 일반 공격 자산으로 임시 대체한 뒤 검증 없이 유지.

## 2. 공격 판정 동기화

필요 능력:

- 준비, 판정, 회복 키포즈 분리.
- 근접 접촉, 투사체 생성, 마법 방출 이벤트 지정.
- 공격속도 변화와 취소 규칙 처리.
- 판정 데이터와 표현 데이터의 책임 분리.

산출물:

```text
state_id
playback_duration_or_fps
impact_event
projectile_spawn_event
cancel_window
recovery_duration
vfx_event
audio_event
```

검수 질문:

- 피해가 보이는 접촉보다 먼저 발생하지 않는가?
- 공격속도가 빨라져도 준비와 회복의 의미가 남는가?
- 판정 전·후 취소 결과가 일관적인가?

## 3. 이동과 무게 표현

필요 능력:

- 코드 이동과 시각 보행의 분리.
- 이동속도 변화에 맞는 재생속도·보폭 조정.
- 인간형, 기병, 비행, 대형 체형의 차별화.
- 진행 방향과 무기 방향 정합성 검수.

실패:

- 발 미끄러짐.
- 대형 유닛을 일반 보행의 느린 재생으로만 표현.
- 기병과 탈것의 주기가 분리됨.

## 4. 피격·사망 피드백

필요 능력:

- 가벼운 피격과 강한 피격을 구분.
- 연속 피격 내부 간격 설계.
- 사망 시 충돌·공격·상호작용 해제 시점 정의.
- 대형 붕괴와 통로·위험 영역의 관계 표현.

검수 질문:

- 다수 공격에서 유닛이 계속 떨리지 않는가?
- 사망 자산이 이동로나 목표 판정을 막지 않는가?
- 사망 연출이 추가 공격 전조로 오인되지 않는가?

## 5. 다수 유닛 연출

필요 능력:

- 결정론적 루프 시작 오프셋.
- 동시 VFX·오디오 밀도 제한.
- 화면 흔들림과 히트 스톱의 지역·강도 제어.
- 다른 전선과 UI 가독성 보존.

실패:

- 모든 유닛이 같은 프레임으로 움직여 복제처럼 보임.
- 매 타격마다 전체 화면 정지.
- 중요 전조가 일반 VFX에 묻힘.

## 6. 스킨·진영 변형 재사용

필요 능력:

- 공용 전투·상태·타이밍과 시각 자산 분리.
- 스킨별 프레임 레이아웃·피벗·이벤트 호환성 검증.
- 시각 차이와 규칙 차이를 혼동하지 않음.

산출물:

- 공용 애니메이션 계약.
- 스킨별 Visual Set 목록.
- 프레임·피벗·이벤트 호환성 테스트.

실패:

- 외형만 다른 유닛을 별도 코드·스탯 데이터로 복제.
- 체형 차이가 큰데도 같은 접촉 프레임을 강제.

## 7. 승리·패배 연출

필요 능력:

- 개별 처치, 교전 종료, 목표 확보, 스테이지 종료를 구분.
- 병종별 짧은 승리 정체성 설계.
- 결과 UI, 오디오, 카메라의 순서 제어.
- 반복 피로와 조작 잠금 최소화.

검수 질문:

- 전투 중 작은 성공마다 흐름이 멈추지 않는가?
- 최종 승리가 충분히 보상적으로 느껴지는가?
- 결과 화면까지 기다리는 시간이 과도하지 않은가?

## 8. 실제 화면 검증

- 최소 전략 줌.
- 같은 병종 5기 이상 겹침.
- 다른 병종·진영과 혼전.
- HUD와 위험 전조 포함.
- 속도 버프·감속·일시정지.
- 최대 객체 수와 저사양 조건.
- 같은 시드 재현.

## 9. 완료 기준

- 상태와 자산 누락을 데이터 검증에서 탐지한다.
- 공격 판정과 시각 이벤트 오차가 목표 범위 안이다.
- 역할별 이동·공격 무게가 실제 화면 크기에서 구분된다.
- 연속 피격과 다수 공격에서도 상태가 안정적이다.
- 스킨·진영 변형이 공용 계약을 지킨다.
- 승리·패배 연출이 다음 UI와 입력을 지연하지 않는다.

## 10. 작업 프롬프트에 포함할 항목

```md
- 기준 카메라와 실제 화면 크기
- 역할별 필수 상태
- 공격 데이터와 이벤트 소유자
- 이동속도·공격속도 범위
- 프레임·방향·피벗 규격
- 스킨·진영 재사용 구조
- 카메라·VFX·오디오 제한
- 완료 기준과 재현 가능한 테스트
```
