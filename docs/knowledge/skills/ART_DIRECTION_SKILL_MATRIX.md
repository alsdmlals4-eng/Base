# 아트 디렉션 실무 스킬 매트릭스

- 상태: 공용 스킬 계약
- 목적: 아트 기획·검수 작업에 필요한 입력, 산출물, 판단 능력과 실패 기준을 명확히 한다.

## 1. 입력 읽기

필수 입력:

- 제품 한 문장과 핵심 사용자 경험.
- 플랫폼, 해상도, 카메라 거리.
- 화면에 동시에 나타나는 객체 수.
- 역할·진영·등급·상태 목록.
- UI와 VFX가 차지하는 영역.
- 제작 방식, 인력, 일정, 예산.
- 기존 자산과 참고 사례.

좋은 수행:

- 스타일 키워드보다 실제 화면 제약을 먼저 요약한다.
- 확정, 가설, 미확정을 분리한다.
- 기존 자산을 폐기하기 전에 재사용 가능성을 확인한다.

실패:

- 화면 크기와 카메라를 확인하지 않고 고해상도 콘셉트부터 제안.
- 프로젝트의 핵심 역할과 상관없는 장식 중심 무드보드.

## 2. 형태 언어 설계

필요 능력:

- 역할을 실루엣, 무기 길이, 자세, 몸통 덩어리로 변환.
- 진영 차이를 선, 대칭성, 소재, 부속 구조로 정의.
- 등급을 단순 색상과 크기 외의 구조적 변화로 표현.
- 건물 기능을 큰 형태와 부속 설비로 구분.

산출물:

- 형태 언어 표.
- 병종·건물 실루엣 카드.
- 금지 예시.
- 흑백 축소 비교.

검수 질문:

- 색을 제거해도 역할이 남는가?
- 같은 등급 여러 기가 모여도 읽히는가?
- 장식이 무기와 상태를 가리지 않는가?

## 3. 팔레트·재질·광원

필요 능력:

- 핵심·보조 팔레트 분리.
- 반복 재질의 명암 단계 통일.
- 환경, 캐릭터, 위험 전조의 대비 우선순위 설계.
- 색각 이상과 저해상도 조건 고려.

산출물:

- 팔레트 표.
- 재질 샘플.
- 광원 방향과 발광 사용 규칙.
- 배경 위 가독성 테스트.

실패:

- 모든 중요 정보를 색상 하나에 의존.
- 상시 발광으로 스킬 전조와 등급 연출을 무력화.

## 4. 제작 규격화

필요 능력:

- 화면 크기에서 역산한 캔버스·픽셀·피벗 규격.
- 방향 수, 프레임 수, 아틀라스 구조 결정.
- 상태·등급·파손 변형의 재사용 구조 설계.
- 파일명과 버전 규칙 작성.

산출물:

- 자산 명세 표.
- 제작 우선순위.
- 스프라이트·모델·UI 파일 구조.
- 완료·검수 체크리스트.

검수 질문:

- 제작자가 추가 질문 없이 다음 자산을 만들 수 있는가?
- 같은 자산을 다른 화면과 상태에 재사용할 수 있는가?
- 파일 구조가 코드·데이터 계약과 맞는가?

## 5. 실제 화면 검수

필요 능력:

- 원화보다 실제 플레이 화면에서 평가.
- 최소·기본·최대 줌 비교.
- 다수 겹침, HUD, VFX, 배경과 함께 검수.
- 저해상도, 흑백, 색각, 움직임 상태 확인.

권장 검수 순서:

```text
실루엣
→ 실제 크기
→ 다수 겹침
→ HUD·VFX 포함
→ 상태·등급 변형
→ 애니메이션
→ 성능
```

## 6. 생성형 콘셉트 감독

필요 능력:

- 탐색용과 최종 자산을 구분.
- 관절, 손, 무기, 문자, 광원 오류 검수.
- 기존 IP와 유사한 고유 디자인 제거.
- 픽셀·리깅·모델링 제작 규격으로 재해석.

실패:

- 생성 결과를 그대로 납품.
- 출처·권리·후처리 책임 미기록.

## 7. 작업 완료 기준

- 실제 화면 크기에서 역할·진영·상태를 읽을 수 있다.
- 형태 언어와 제작 규격이 문서화되어 있다.
- 최소 한 개 대표 자산을 실제 HUD와 배경 위에서 검증했다.
- 자동 탈락 조건과 수정 경로가 있다.
- 미완성·임시·최종 자산이 구분된다.
- 검증하지 않은 품질을 완료로 보고하지 않는다.

## 8. 작업 프롬프트에 포함할 항목

```md
- 제품과 플레이어 경험
- 기준 해상도·카메라·화면 크기
- 역할·진영·등급·상태
- 기존 자산과 참고 문서
- 포함·제외 범위
- 제작 방식과 예산
- 필요한 산출물
- 실제 화면 검수 기준
- 저작권·생성형 도구 경계
```
