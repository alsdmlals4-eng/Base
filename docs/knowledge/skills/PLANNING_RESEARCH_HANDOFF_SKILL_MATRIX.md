# 기획·조사·인수인계 스킬 매트릭스

- 상태: 공용 스킬 계약
- 목적: 기획 작업에 필요한 능력을 도구 이름이 아니라 사용 조건, 입력, 과정, 산출물, 실패 기준으로 정의한다.

외부 플러그인·MCP·에이전트는 아래 능력을 보조하는 선택 수단이다. 설치 여부와 브랜드가 결과 품질을 보장하지 않는다.

## 1. 스킬 지도

| 스킬 | 사용 조건 | 핵심 산출물 |
|---|---|---|
| 의도 합성 | 요청이 모호하거나 목표가 충돌함 | 문제·목표·사용자 가치·제외 범위 |
| 현재 상태 감사 | 기존 프로젝트·문서를 인수함 | 구현 사실·승인 계획·차이표·위험 경로 |
| 기획 구조화 | 아이디어를 실행 가능한 체계로 전환 | 방향서·분야별 책임 문서·로드맵 |
| 콘텐츠 정체성 설계 | 사건·장면·기능을 추가함 | 한 줄 약속·핵심 동사·PoC·차별점 |
| 서사·관계 설계 | 캐릭터·선택·후속 반응 작성 | 장면 비트·목소리 카드·선택 기억 |
| 아트 디렉션 | 시각 스타일과 자산 기준 수립 | 시각 약속·형태·색·재료·자산 티어·QA |
| 프레젠테이션 설계 | 대화 UI·표정·컷인·음향 구성 | 시선 흐름·의미 키·폴백·접근성 |
| 벤치마킹 | 새로운 문제 해결 근거 필요 | 관찰·적용·제외·검증 카드 |
| 최신 정보 검증 | 제품·기술·시장·정책이 변할 수 있음 | 출처·확인일·불확실성·결론 |
| 수직 PoC 설계 | 큰 계획의 위험 가정 검증 | 최소 진입·행동·결과·복귀·테스트 |
| 수용 기준 작성 | 구현 완료 판단 필요 | 조건→행동→관찰 결과 |
| 인수인계 압축 | 담당자·AI·계정이 바뀜 | 읽기 순서·책임 원본·다음 행동 |
| 회고·지식 승격 | 작업 종료 후 교훈 축적 | 실패 원인·재발 방지·공용 사례 후보 |

## 2. 의도 합성

### 입력

- 사용자 요청.
- 문제 배경.
- 기존 결정.
- 품질·일정·비용 우선순위.

### 과정

1. 결과물보다 해결할 문제를 찾는다.
2. 원하는 행동·감정·품질을 추출한다.
3. 충돌하는 목표를 표시한다.
4. 확인된 사실과 추정을 분리한다.
5. 불필요한 질문과 임의 확장을 피한다.

### 산출물

```text
목표
사용자 가치
현재 문제
포함 범위
제외 범위
결정 필요 항목
```

### 실패 기준

- 요청 문장을 그대로 길게 반복함.
- 사용자 가치 없이 산출물 형식만 정함.
- 확인 없이 구현 세부를 확정함.

## 3. 현재 상태 감사

### 입력

- 저장소·문서·데이터·실행 화면.
- 최근 커밋·Issue·QA.
- 전달 패키지와 외부 산출물.

### 과정

- 구현, 승인 계획, 진행 중, 백업을 구분한다.
- 실제 파일과 문서의 차이를 표로 만든다.
- 보호할 ID·저장·상태 소유자를 확인한다.
- 오래된 자료와 현행 원본을 분리한다.
- 외부 산출물을 신뢰하지 않는 입력으로 취급한다.

### 산출물

- 기준선.
- 차이표.
- 위험 경로.
- 사용할 책임 원본.
- 미확인 항목.

### 실패 기준

- 문서 존재를 구현 완료로 판단함.
- 과거 QA를 현재 동작 증거로 사용함.
- 사용자 변경과 dirty 상태를 덮어씀.

## 4. 기획 구조화

### 입력

- 합의된 목표와 사용자 가치.
- 현재 상태 감사 결과.
- 프로젝트 제약과 책임 문서.

### 과정

```text
현재 상태
→ 프로젝트 방향
→ 전체 설계
→ 분야별 책임 문서
→ 실행 로드맵
→ 구현 지시
→ 테스트·QA
→ 사례·회고
```

같은 규칙을 여러 문서에 복사하지 않고 질문별 책임 원본을 정한다.

### 산출물

- 문서 역할표.
- 읽기 순서.
- 상태 표기.
- 단계 의존성.
- 갱신 책임.

## 5. 서사·관계 설계

### 핵심 능력

- 장면의 중심 질문을 하나로 제한.
- 행동만으로 흐름을 먼저 작성.
- 인물별 욕망·두려움·판단 기준 정의.
- 기능·관계·세계관 대화의 밀도 분리.
- 선택을 후속 말투·정보 공유·장면 해금으로 연결.
- 설명을 갈등·행동·소품으로 변환.
- 정보 누설·반복·분량·말투 검수.

### 산출물

- 장면 비트 시트.
- 캐릭터 목소리 카드.
- 선택·기억·후속 반응 매트릭스.
- 데이터·연출 경계.

### 검증

- 대사를 가려도 행동 흐름을 이해하는가?
- 말투로 인물을 구분하는가?
- 선택이 다음 장면에 기억되는가?
- 반복 재생과 저장 복구에서 중복이 없는가?

## 6. 아트 디렉션

### 핵심 능력

- 시각 약속 문장 작성.
- 형태·색·명암·재료·질감 축 분해.
- 레퍼런스 보드의 관찰 요소 라벨링.
- 캐릭터·대상 실루엣과 상태 차별화.
- 자산 T0~T3 우선순위.
- 생성형 이미지 프롬프트·manifest 설계.
- 실제 화면 크롭·알파·텍스트 없음 QA.

### 산출물

- 아트 방향서.
- 캐릭터·대상 디자인 카드.
- 자산 목록과 티어.
- 제작 브리프와 QA 체크.

### 실패 기준

- 작품 이름만 레퍼런스로 적음.
- 실제 화면 크기와 UI 조합을 검증하지 않음.
- 생성 이미지 글자를 의미 텍스트로 사용함.
- 폴백과 우선순위가 없음.

## 7. 프레젠테이션 설계

### 핵심 능력

- 첫 시선과 중심 질문 정의.
- 장면 비트와 카메라 목적 분해.
- 표정 의미 키·폴백 설계.
- 컷인 유형·빈도·타이밍 제한.
- 음향·색상과 동등한 텍스트 신호.
- 모션 감소·점멸 제한·포커스 검수.
- 표현 계층과 상태 로직 분리.

### 산출물

- 화면 시선 흐름.
- 이벤트 비트 시트.
- 표정·컷인 레지스트리.
- 접근성 폴백.
- 입력·상태 소유 계약.

## 8. 벤치마킹과 최신 정보 검증

### 과정

1. 결정할 질문을 한 문장으로 적는다.
2. 사례와 출처 수준을 정한다.
3. 직접 관찰과 1차 출처를 우선한다.
4. 관찰·해석·결정을 분리한다.
5. 적용·조건부 적용·제외를 기록한다.
6. 최신성이 필요한 사실은 확인일을 남긴다.
7. PoC와 사용자 테스트로 결론을 갱신한다.

### 산출물

- 근거 표.
- 사례 카드.
- 적용 결론.
- 불확실성과 추가 검증.

### 실패 기준

- 작품 목록과 스크린샷만 수집함.
- 검색 요약만 보고 결론을 씀.
- 사용자 반응과 제품 사실을 혼합함.
- 외형과 고유 문구를 복제함.

## 9. 수직 PoC

### 최소 단위

```text
진입 조건
→ 사용자 행동 1~3개
→ 성공 결과 1개
→ 실패 결과 1개
→ 기록 또는 상태 변화 1개
→ 복귀·저장 검증
```

### 질문

- 핵심 행동을 이해하는가?
- 다시 하고 싶어 하는가?
- 성공·실패 이유를 설명하는가?
- 주변 시스템 없이 가치가 성립하는가?
- 전체 제작 전에 가장 위험한 가정을 검증했는가?

## 10. 수용 기준 작성

`조건 → 행동 → 관찰 결과` 형식으로 쓴다.

예시:

```text
요청한 표정 자산이 없을 때
→ 장면을 재생하면
→ 기본 표정으로 폴백하고 대사·입력·저장 상태가 유지된다.
```

“멋지다”, “자연스럽다”, “최적화됐다” 같은 표현은 관찰 가능한 결과로 바꾼다.

## 11. 인수인계 압축

### 과정

- 먼저 읽을 문서를 3~6개로 제한한다.
- 상태와 계획을 분리한다.
- 분야별 책임 원본과 실제 파일을 연결한다.
- 다음 작업의 진입·위험·완료 기준을 적는다.
- 오래된 자료를 기본 읽기에서 제외한다.
- 정확한 과거 원문 경로를 보존한다.

### 산출물

- Handoff 또는 Active Context.
- 문서 역할표.
- 현재 실행 순서.
- 보호 계약.
- 미확인·미검증 목록.

## 12. 회고와 지식 승격

작업 종료 후 다음을 기록한다.

- 발생한 실수와 누락.
- 원인.
- 재발 방지 체크.
- 다음 프롬프트에 넣을 문장.
- 프로젝트 전용으로 남길 내용.
- Base 사례로 일반화할 후보.
- 반복 검증 후 method·skill로 승격할 후보.

## 13. 도구 선택

한 작업에서 모든 도구를 사용하지 않는다.

```text
현재 문제
→ 필요한 능력 1~3개
→ 최소 권한의 도구
→ 산출물과 검증
→ 회고
```

외부 도구 사용 전에는 권한, 외부 API, 비밀값, 쓰기 범위, 실패 복구, 독립 검증 가능성을 확인한다.

## 14. 스킬 카드 양식

```md
## 스킬
- 이름:
- 해결하는 문제:
- 사용 조건:
- 입력:
- 수행 단계:
- 산출물:
- 검증:
- 권한·비용·위험:
- 사용하지 않을 조건:
- 관련 사례:
- 상태: 가설 / 패턴 / 원칙
```
