# AI Workflow Checklist

이 문서는 여러 프로젝트에서 공통으로 사용할 작업 순서와 체크리스트다.

## 1. 표준 작업 순서

```text
0. 작업 유형 분류
1. 사용자 최신 지시 확인
2. Base 공용 규칙 확인
3. 프로젝트 전용 규칙 확인
4. 현재 Issue / MVP 기준선 확인
5. 작업 준비 상태 체크
6. 벤치마킹 필요 여부 판단
7. 구현 범위 / 제외 범위 확정
8. 위험 분석 / 영향 파일 예상
9. GitHub Issue 작성 또는 수정
10. HTML 전체 흐름 대시보드 갱신
11. Codex Goal 작성
12. 구현
13. 실행 검수
14. 결과 보고 / 회고
15. 공용 규칙 승격 여부 판단
```

## 2. 작업 유형 분류

| 유형 | 설명 | 경로 |
|---|---|---|
| 빠른 수정 | 오탈자, 단순 문서, 단일 데이터 | 빠른 경로 |
| 기능 작업 | 새 기능, 여러 파일, 저장/불러오기 | 기능 경로 |
| 기획 작업 | 시스템/루프/플레이어 경험 설계 | 기획 경로 |
| 검수 작업 | Codex 결과 확인, 오류 분석 | 검수 경로 |
| 규칙 정리 | 반복 규칙 문서화 | 문서 경로 |

## 3. 작업 시작 체크리스트

```md
- [ ] 사용자의 최신 지시를 확인했는가?
- [ ] 작업 유형을 분류했는가?
- [ ] Base 공용 규칙과 프로젝트 전용 규칙을 구분했는가?
- [ ] 현재 Issue가 있는가?
- [ ] 이전 작업과 다음 작업이 연결되는가?
- [ ] 벤치마킹이 필요한 작업인가?
- [ ] 구현 AI와 기획 AI의 역할이 분리되어 있는가?
- [ ] 대시보드/문서/구현 범위가 섞이지 않았는가?
```

## 4. 좋은 프롬프트 변환 체크리스트

```md
- [ ] 목표
- [ ] 배경
- [ ] 사용자의 의도
- [ ] 플레이어 경험
- [ ] 구현 범위
- [ ] 제외 범위
- [ ] 데이터 요구사항
- [ ] 엔진/씬/노드/파일 영향
- [ ] 완료 기준
- [ ] 테스트 방법
- [ ] 결과물 형식
```

## 5. 벤치마킹 체크리스트

```md
- [ ] 유사/인접 사례를 확인했는가?
- [ ] 게임/사례를 적었는가?
- [ ] 관찰포인트를 적었는가?
- [ ] 반응을 적었는가?
- [ ] 프로젝트 적용 이유를 적었는가?
- [ ] 반영할 것과 제외할 것을 나눴는가?
- [ ] 출처 없는 일반론으로 판단하지 않았는가?
```

권장 표:

```md
| # | 게임/사례 | 관찰포인트 | 반응 | 프로젝트 적용(+사유) |
|---:|---|---|---|---|
```

## 6. Issue 체크리스트

```md
- [ ] 목표
- [ ] 핵심 흐름
- [ ] 벤치마킹 반영/제외 결론
- [ ] 구현 범위
- [ ] 제외 범위
- [ ] 완료 기준
- [ ] 작업 전 확인 파일
- [ ] 공통 규칙 참조
- [ ] 작업 후 보고 형식
```

## 7. Codex Goal 체크리스트

```md
- [ ] Issue 번호가 맞는가?
- [ ] Base 공용 규칙과 프로젝트 전용 규칙을 참조하는가?
- [ ] 구현 범위만 들어갔는가?
- [ ] 제외 범위가 있는가?
- [ ] 완료 기준이 있는가?
- [ ] 테스트 기준이 있는가?
- [ ] 작업 후 보고 형식이 있는가?
- [ ] 대시보드나 긴 벤치마킹 전문이 빠져 있는가?
```

## 8. 결과 검수 체크리스트

```md
- [ ] 변경 파일이 보고되었는가?
- [ ] 변경 이유가 설명되었는가?
- [ ] 구현 내용이 Issue와 일치하는가?
- [ ] 기존 흐름이 깨지지 않았는가?
- [ ] 저장/불러오기를 확인했는가?
- [ ] 사용자가 실행할 확인 순서가 있는가?
- [ ] 남은 위험이 적혀 있는가?
- [ ] 사용 도구와 미사용 도구가 사실대로 적혀 있는가?
```

## 9. 회고 체크리스트

```md
- [ ] 반복된 혼동이 있었는가?
- [ ] 다음 작업에서 줄일 규칙이 있는가?
- [ ] 공용 규칙으로 승격할 내용이 있는가?
- [ ] 프로젝트 전용으로 남길 내용이 있는가?
```
