# 공용 연출·프레젠테이션 기획 방법

> 문서 위치: `docs/planning/PRESENTATION_DIRECTION_METHOD.md` | 아트 방향: `docs/planning/ART_DIRECTION_METHOD.md` | 사례: `docs/planning/CASE_LIBRARY.md`

## 1. 목적

화면 전환, 카메라, 캐릭터 배치, 표정, 컷인, VFX, 음향, 대사창을 따로 설계하지 않고 **정보 위계와 감정 변화**를 전달하는 하나의 프레젠테이션 체계로 만든다.

연출은 내용을 대신 결정하지 않는다. 게임·앱·인터랙티브 콘텐츠의 실제 상태를 읽어 보여주며, 연출이 꺼지거나 자산이 누락돼도 핵심 정보와 입력이 유지돼야 한다.

## 2. 연출의 네 역할

1. **방향 제시:** 지금 어디를 보고 무엇을 해야 하는지 알려준다.
2. **중요도 표시:** 여러 정보 중 무엇이 핵심인지 구분한다.
3. **감정 강화:** 인물·상황·결과의 정서를 더 잘 느끼게 한다.
4. **기억 형성:** 능력, 관계, 규칙, 성취의 결정적 순간을 남긴다.

역할이 없는 효과는 줄이거나 제거한다.

## 3. 화면 시선 설계

한 장면에는 하나의 첫 시선과 하나의 중심 질문을 둔다.

권장 순서:

```text
공간과 현재 상태
→ 주요 대상 또는 인물
→ 현재 행동·대사·질문
→ 선택 또는 조작
→ 보조 기록·메뉴
```

화면 구성 전에 다음을 적는다.

- 첫 1초에 보여야 할 것
- 3초 안에 이해할 것
- 사용자가 선택 전에 읽어야 할 정보
- 선택 후 즉시 변해야 할 요소
- 상세 정보가 숨겨질 위치

## 4. 이벤트 비트

장면은 필요에 따라 다음 비트를 조합한다.

```text
공간 제시
→ 대상 진입 또는 변화
→ 핵심 정보·대화
→ 작은 행동·소품 변화
→ 사용자 선택·조작
→ 즉시 시각·음향 반응
→ 결과·기록
→ 다음 질문 또는 여운
```

비트마다 목적이 없으면 합치거나 삭제한다.

## 5. 카메라와 배치

### 카메라 질문

- 정보 확인인가, 감정 확인인가?
- 누가 주도하고 누가 반응하는가?
- 사용자 입력 전에 필요한 공간 정보가 있는가?
- 가까운 컷이 실제로 새 정보를 보여주는가?

### 기본 배치

- 1인: 공간과 행동을 함께 보여준다.
- 2인: 주도자와 반응자의 시선·거리로 관계를 표현한다.
- 3인: 주 화자 1명과 반응자 2명으로 우선순위를 둔다.
- 4인 이상: 전원을 작게 배치하지 않고 컷 전환·그룹 반응으로 나눈다.

### 거리 의미

- 가까워짐: 신뢰, 압박, 보호, 위협 중 무엇인지 맥락으로 구분한다.
- 멀어짐: 경계, 단절, 공간적 위험, 정보 격리.
- 가림: 권력, 비밀, 보호, 시야 제한.

## 6. 표정 체계

표정은 파일 번호보다 의미 키로 요청한다.

예시 키:

- `neutral`
- `focus`
- `alert`
- `anger`
- `concern`
- `soft`
- `awkward`
- `hurt`
- `resolve`
- `special_state`

캐릭터마다 모든 키를 만들 필요는 없다. 공통 의미를 캐릭터 고유 방식으로 표현한다.

### 표정 사용 규칙

- 대사와 같은 감정을 반복 설명하지 않는다.
- 말과 속마음이 다를 때 가장 가치가 높다.
- 표정 변화 빈도를 제한해 작은 차이가 읽히게 한다.
- 자산이 없으면 가까운 감정군 또는 기본 표정으로 폴백한다.
- 표정 부재가 대사·입력·진행을 막지 않는다.

## 7. 컷인 체계

컷인은 일반 화면을 잠시 재구성해 중요 순간을 강조한다.

공용 유형 예시:

| 유형 | 사용 목적 |
|---|---|
| 능력·기능 발동 | 누가 어떤 방식으로 기여했는지 표시 |
| 위험 경고 | 즉시 주의할 변화와 행동 중단 요구 |
| 규칙·가설 확인 | 사용자의 해석이 맞았음을 현장에서 입증 |
| 관계 반응 | 선택 기억과 감정 전환 강조 |
| 특수 상태 | 오염, 오류, 변형, 감응 등 기본 규칙 이탈 |
| 기록·성취 갱신 | 사용자가 만든 결과가 남았음을 표시 |

### 컷인 제한

- 일반 대사마다 사용하지 않는다.
- 같은 유형을 짧은 간격으로 반복하지 않는다.
- 선택 입력 중 컷인으로 포커스를 빼앗지 않는다.
- 컷인이 사라져도 텍스트·아이콘·상태 변화로 의미를 이해할 수 있어야 한다.
- 장르와 맞지 않는 과도한 축하 연출을 피한다.

## 8. 전환과 타이밍

### 전환 목적

- 장소 변화
- 시간 변화
- 정보 단계 변화
- 감정 단계 변화
- 사용자 통제의 시작·종료

전환이 단순 장식이면 줄인다.

### 타이밍 기준

- 일반 UI 반응: 입력 직후
- 짧은 캐릭터 반응: 0.2~0.8초
- 컷인: 약 0.8~2초
- 사용자가 읽어야 할 핵심 문구: 자동 소거 금지 또는 충분한 확인 시간
- 위험 경고: 빠르되 문구와 아이콘을 동시에 제공

정확한 수치는 프로젝트에서 테스트한다.

## 9. 음향과 시각 신호

- 중요한 상태는 소리만으로 전달하지 않는다.
- 캐릭터별 전체 음성보다 반복 가능한 신호음·장비음·환경음을 우선할 수 있다.
- 성공·실패 음향은 결과의 성격과 장르 톤에 맞춘다.
- 공포·충격 효과는 빈도와 음량을 제한한다.
- 반복 알림은 짧게, 최초 안내는 충분히 설명한다.

## 10. 텍스트 연출

- 흔들림·왜곡·색 변화는 핵심 단어 일부에 제한한다.
- 읽기 속도를 방해하는 타이핑 효과는 끌 수 있어야 한다.
- 자동 진행은 선택, 경고, 중요한 기록 갱신에서 멈춘다.
- 이름표, 대사, 시스템 문구의 역할을 시각적으로 구분한다.
- 생성 이미지 안의 텍스트를 의미 전달 수단으로 사용하지 않는다.

## 11. 접근성

### 필수 폴백

- 모션 감소
- 점멸 제한
- 색상 독립 아이콘·문구
- 텍스트 우선 정보
- 읽기 시간 보장
- 키보드 포커스와 Esc/닫기 일관성
- 자막과 음향 설명 필요성 검토

### 모션 감소 시

- 카메라 줌을 짧은 페이드나 강조 테두리로 대체
- 화면 흔들림을 아이콘·문구·색 변화로 대체
- 이동 애니메이션 시간을 줄이거나 즉시 배치
- 핵심 의미는 동일하게 유지

## 12. 상태 소유 경계

연출 계층은 다음을 할 수 있다.

- 현재 상태에 맞는 표정·자산 선택
- 컷인과 전환 재생
- 입력 잠금과 해제
- 텍스트·아이콘·음향 표현
- 폴백 선택과 개발 로그 기록

연출 계층은 다음을 해서는 안 된다.

- 결과 판정
- 보상 지급
- 관계 변화 결정
- 퀘스트 완료 처리
- 저장 버전 변경
- 대사 분기 조건의 중복 정의

상태 변화는 도메인 로직이 결정하고, 연출은 결과를 표현한다.

## 13. 목업과 실제 구현

목업은 방향 참고이며 다음을 반드시 실제 화면에서 재검증한다.

- 텍스트 길이
- 해상도와 화면비
- 마우스·키보드·패드 입력
- 포커스 순서
- 자산 크롭
- 실제 데이터 개수
- 저장·복귀 중간 상태
- 옵션과 접근성

목업의 픽셀 배치를 그대로 복제하는 것보다 시선 우선순위와 역할을 보존한다.

## 14. 구현 절차

1. 현재 화면과 상태 소유자를 감사한다.
2. 한 장면의 첫 시선·중심 질문·입력 순서를 적는다.
3. 기본 텍스트·UI만으로 흐름을 완성한다.
4. 표정 의미 키와 기본 폴백을 연결한다.
5. 가장 가치가 큰 컷인 1종을 추가한다.
6. 접근성 폴백을 같은 단계에서 만든다.
7. 실제 해상도·입력·복귀를 검증한다.
8. 안정된 패턴만 다른 장면으로 확장한다.

## 15. 검수 체크

- [ ] 첫 시선과 중심 질문이 명확하다.
- [ ] 현재 화자·주요 대상·선택이 혼동되지 않는다.
- [ ] 표정·컷인이 의미 있는 순간에만 사용된다.
- [ ] 연출을 꺼도 핵심 정보를 이해한다.
- [ ] 자산 누락 시 기본 폴백으로 진행된다.
- [ ] 컷인·오버레이·메뉴의 입력 포커스가 충돌하지 않는다.
- [ ] 긴 텍스트와 최소 해상도에서 잘림이 없다.
- [ ] 음향·색상만으로 상태를 전달하지 않는다.
- [ ] 프레젠테이션 계층이 결과와 저장 상태를 소유하지 않는다.
- [ ] 실제 사용자가 중요한 순간을 더 잘 기억하는지 확인했다.
