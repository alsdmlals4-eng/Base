# 질문·출처 모델

## 출처 유형

| 값 | 의미 | 요구사항 확정 권한 |
|---|---|---|
| `user_stated` | 사용자가 직접 말하거나 승인한 의도·결정 | 있음 |
| `repository_observed` | 코드·문서·자산·테스트에서 확인한 현재 사실 | 현재 상태에만 있음 |
| `reference_derived` | 외부 저장소·벤치마크·문헌에서 얻은 후보 | 없음, 사용자 확인 필요 |
| `model_inferred` | AI가 맥락에서 추론한 후보 | 없음, 사용자 확인 필요 |

현재 사실과 원하는 미래를 혼동하지 않는다. 저장소가 어떤 구조라는 사실은 그 구조를 유지하라는 요구가 아니며, 참고 작품이 성공했다는 사실은 복제 승인도 아니다.

## 질문 분류

1. 저장소 사실: 먼저 조사하고 결과를 제시한다.
2. 사용자 판단: 목표·선호·비용·위험·허용할 트레이드오프를 묻는다.
3. 기술 판단: 사용자 결과와 제약을 먼저 확인한 뒤 구현자가 제안한다.
4. 보류 가능: 선행 결정·실험·데이터가 필요하면 조건과 재개 시점을 기록한다.

## 균형 축

- 목표·플레이어/사용자 경험
- 범위·비목표
- 제약·비용·호환성
- 산출물 종류와 품질
- 완료 기준·검증
- 의존성·보호 대상·롤백

몇 차례 한 축만 다뤘다면 다른 미해결 축으로 돌아간다. 질문 수를 채우기 위해 묻지 않으며, 답이 실행 계약을 바꿀 때만 묻는다.

## 참고자료 대조

참고자료마다 다음을 구분한다.

- 채택할 원리
- 프로젝트 상황에 맞게 변형할 부분
- 제외할 정체성·표현·프로젝트 전용 코드
- 우리 프로젝트가 달라야 하는 지점
- 아직 확인하지 못한 가정
