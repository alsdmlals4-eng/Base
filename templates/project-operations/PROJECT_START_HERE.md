## 새 작업자의 읽기 순서

```text
프로젝트 AGENTS.md
→ 이 START_HERE.md
→ ACTIVE_CONTEXT.md
→ DOCUMENTATION_MAP.md
→ 현재 작업의 분야별 본책
→ Roadmap·Issue·Goal·Plan
→ 실제 코드·데이터·자산·테스트
```

Base 공용 기준이 필요하면 프로젝트 `BASE_RULES_VERSION.md`에 기록된 커밋과 Base `START_HERE.md`를 확인한다.

## 지금 하지 말아야 할 것

- 보류·폐기 영역을 승인 없이 구현하지 않는다.
- 기존 승인 이미지가 있는 항목의 새 시안을 별도 지시 없이 만들지 않는다.
- 문서가 있다는 이유로 구현·검증 완료로 표시하지 않는다.
- 범위 밖 리팩터링과 기능 확장을 현재 작업에 섞지 않는다.
- `v2`, `final`, `latest` 활성 복제본을 만들지 않는다.

## 콜드 스타트 확인

새 작업자는 10분 안에 다음을 답할 수 있어야 한다.

- 게임의 핵심 약속은 무엇인가?
- 현재 단계와 다음 게이트는 무엇인가?
- 지금 가장 먼저 할 작업은 무엇인가?
- 무엇이 확정·구현·검증·미확정인가?
- 최신 이미지와 실제 캡처는 어디인가?
- 각 분야의 현행 책임 원본은 어디인가?
- 실제 코드·데이터·테스트는 어디인가?

답하지 못하면 이 문서 또는 연결된 책임 원본을 갱신한다.
