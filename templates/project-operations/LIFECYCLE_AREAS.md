## 1. `[현행]`

현재 프로젝트가 실제로 참조하는 책임 원본, 프로젝트 스킬과 최신 파생본이다.

운영 규칙:

- 한 질문에 현행 책임 원본 하나
- 새 결정은 기존 현행 원본 갱신
- 기본 작업 컨텍스트는 현재 작업에 필요한 현행 원본만 선택
- 상태·경로·갱신 트리거를 Documentation Map에 등록

## 2. `[백업]`

Git 이력만으로 충분하지 않은 외부 원본·승인·감사·법무·비교 자료를 보존한다.

사용 조건:

- 사용자가 별도 보존 요청
- 외부 링크가 고정 경로 요구
- 원본 파일 자체가 승인·출처 증거
- 감사·법무·비교 검토에 완전한 패키지 필요

사용하지 않는 조건:

- 단순 이전 버전
- `v2`, `final`, `latest`, 날짜별 복제본
- 현행 문서에 이미 완전히 반영된 일반 초안

| 자료 | 보존 이유 | 원본·출처 | 외부 참조 | 기본 읽기 제외 | 제거 조건 |
|---|---|---|---|---|---|
|  |  |  |  | 예 |  |

## 3. `[보류]`

현재 구현·활성 범위가 아니지만 재개 가치가 있는 내용을 보존한다.

각 항목 필수 정보:

```yaml
item:
hold_reason:
resume_condition:
related_owner_document:
prerequisites:
not_in_current_scope: true
last_reviewed:
```

| 항목 | 보류 이유 | 재개 조건 | 책임 원본 | 선행 작업 | 현재 범위 제외 | 마지막 검토 |
|---|---|---|---|---|---|---|
|  |  |  |  |  | 예 |  |

별도 재개 결정 전에는 구현하지 않는다.

## 4. `[제거 후보]`

현행 원본에 완전히 흡수됐을 가능성이 있으나 보존·참조 검증과 사용자 승인을 기다리는 자료다.

| 파일 | 흡수된 현행 원본 | 고유 정보 확인 | 참조 확인 | 보류·미검증 없음 | 복구 방법 | 승인 |
|---|---|---|---|---|---|---|
|  |  |  |  |  | Git 이력/백업 |  |

모든 조건이 확인되기 전에는 삭제하지 않는다.

## 5. 기본 읽기 제외

다음 영역은 사용자가 재개·감사·비교를 요청하지 않는 한 기본 작업 컨텍스트에서 제외한다.

- `[백업]`
- `[보류]`
- `[제거 후보]`
- archive·hold·deprecated
- 과거 AI 대화 전체
- 실패한 탐색 과정의 전체 로그
- 현행 원본에 반영된 중복 원문

Documentation Map에는 상세 전문이 아니라 위치, 보존 이유와 재개 조건만 기록한다.
