## 2. 상태 정의

| 상태 | 의미 |
|---|---|
| `REFERENCE` | 아이디어나 일부 요소 참고용 |
| `DIRECTION_APPROVED` | 시각 방향 승인 |
| `PRODUCTION_READY` | 제작 규격과 사용 조건 확정 |
| `IMPLEMENTED` | 게임에 실제 적용됨 |
| `VISUALLY_VALIDATED` | 목표 해상도·실제 화면에서 검증됨 |
| `PARTIAL_REFERENCE` | 특정 요소만 계속 참고 |
| `SUPERSEDED` | 새 기준으로 교체됨 |
| `MIGRATION_PENDING` | 인덱스에는 등록됐으나 파일 이전·저장 미완료 |
| `REJECTED` | 현재 방향에서 사용하지 않음 |

한 자료가 여러 상태를 가질 수 있다. 예: `DIRECTION_APPROVED / MIGRATION_PENDING`.

## 9. 자산 유입·교체 절차

```text
원본·출처 확인
→ 기존 동일 항목 검색
→ 참고·비참고 요소 기록
→ 원본·변환본 저장
→ Asset Manifest 등록
→ 캐노니컬 경로 지정
→ 관련 본책·Documentation Map 연결
→ 엔진 적용
→ 실제 캡처·시각 QA
→ 기존 기준 SUPERSEDED 처리
```

기존 승인 이미지가 있으면 사용자 지시 없이 동일 항목의 새 시안을 만들지 않는다.

## 10. 파일명·경로 규칙

권장:

```text
images/canonical/battlefield_overview.webp
images/captures/battlefield_overview_current.webp
images/references/battlefield_reference_source.webp
```

금지 활성 이름:

```text
battlefield_v2.webp
battlefield_final.webp
battlefield_latest.webp
battlefield_new_copy.webp
```

이전 상태는 Git 이력으로 보존한다.

## 11. 시각 QA

- [ ] 목표 해상도와 최소 해상도에서 확인했다.
- [ ] 배경과 중첩 상황에서 실루엣이 읽힌다.
- [ ] 색만으로 상태·진영·위험을 구분하지 않는다.
- [ ] 텍스트와 아이콘이 실제 언어·배율에서 읽힌다.
- [ ] 애니메이션·VFX가 판정과 입력을 가리지 않는다.
- [ ] 모션 감소·플래시 감소 등 접근성 폴백이 있다.
- [ ] Import·필터·압축·피벗이 규격과 일치한다.
- [ ] 실제 게임 캡처가 문서의 구현 상태와 일치한다.
- [ ] 이미지에 없는 내용을 승인된 것으로 추정하지 않았다.
