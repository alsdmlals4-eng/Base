## 3. 이미지 변경 추가 판정

- [ ] 기존 승인 이미지가 있는지 확인했다.
- [ ] 새 시안 생성이 사용자의 명시적 요청 또는 교체 승인 범위인가?
- [ ] 이미지 전체가 아니라 채택·비채택 요소를 기록했다.
- [ ] 캐노니컬 경로를 유지하거나 변경 근거를 기록했다.
- [ ] Asset Manifest 상태와 실제 파일 존재가 일치한다.
- [ ] Visual DNA가 갱신됐다.
- [ ] 콘셉트와 실제 게임 캡처를 구분했다.
- [ ] 골든 스크린샷 영향이 있다면 시각 회귀를 검토했다.

## 4. 개발 단계 변경 추가 판정

| 게이트 변경 | 필수 확인 |
|---|---|
| Prototype 진입·종료 | 가설, 폐기 가능 범위, 관찰 기준 |
| First Playable | 핵심 루프의 처음부터 끝까지 연결, 치명적 막힘 |
| Vertical Slice | 목표 품질, 대표 경험, 아트·사운드·UX·제작 파이프라인 |
| Production | 파이프라인 처리량, 자산 예산, 일정·비용 위험 |
| Alpha | 주요 기능 연결, 전체 진행, 저장·복귀 |
| Feature Complete | 계획 기능 누락, 범위 고정 |
| Content Complete | 출시 콘텐츠 목록·현지화·자산 상태 |
| Beta | 회귀·성능·밸런스·접근성·릴리스 위험 |
| Release Candidate | 치명적 버그, 플랫폼·배포·법무·복구 계획 |

## 5. 작업 종료 체크

- [ ] 실제 결과와 본책의 확정·구현·검증 상태가 일치한다.
- [ ] 주 책임 본책을 갱신했다.
- [ ] 영향 분야 본책의 계약·상태·위험을 확인했다.
- [ ] Active Context와 Roadmap을 확인했다.
- [ ] Documentation Map과 Changelog를 확인했다.
- [ ] Decision Log가 필요한 변경인지 확인했다.
- [ ] 이미지·사운드·데이터 Manifest를 확인했다.
- [ ] 실행한 테스트와 미검증 항목을 분리했다.
- [ ] 다음 작업과 선행 조건을 기록했다.
- [ ] 콜드 스타트 경로가 깨지지 않았는지 확인했다.
