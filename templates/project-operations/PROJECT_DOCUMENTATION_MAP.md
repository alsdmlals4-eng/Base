> 모든 파일을 무작정 읽지 않는다. 질문·작업 유형에 맞는 현행 책임 원본, foundation 스킬, 분야 스킬과 실제 검증 경로만 선택한다.

## 1. 기본 읽기 순서

```text
최신 사용자 지시
→ 프로젝트 AGENTS.md
→ BASE_RULES_VERSION.md와 적용 Base 기준
→ START_HERE.md
→ ACTIVE_CONTEXT.md·HANDOFF.md
→ 이 DOCUMENTATION_MAP.md
→ 관련 분야 본책
→ PROJECT_SKILL_MAP.md의 foundation + 분야 스킬
→ DEVELOPMENT_GATES.md
→ Roadmap·Issue·Goal·Plan
→ 실제 코드·데이터·자산·테스트
```

기본 제외: `[백업]`, `[보류]`, `[제거 후보]`, archive, hold, deprecated.

## 7. 콜드 스타트 검수

새 AI가 10분 안에 다음을 찾을 수 있어야 한다.

- 프로젝트 목적과 핵심 경험
- 현재 구현·검증 상태
- 다음 작업·선행 조건·게이트
- 금지·보호 범위
- 분야별 본책·프로젝트 스킬
- 실제 코드·데이터·자산·테스트
- 승인 이미지와 최신 PDF
- 보류·확인 필요·미검증
- 작업 종료 갱신 대상

답하지 못한 질문은 이 지도 또는 연결된 책임 원본을 보강한다.
