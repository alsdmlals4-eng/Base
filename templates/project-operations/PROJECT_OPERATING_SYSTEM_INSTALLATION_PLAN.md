# 프로젝트 저장소 운영체계 설치 계획

- 프로젝트:
- 대상 저장소:
- 작성일:
- 기준 브랜치·커밋:
- Base 기준 커밋:
- 프로젝트 유형: `신규 / 운영 중 기존 프로젝트`
- 설치 수준: `Audit only / Governance foundation / Approved migration / Enforcement`
- 상태: `감사 중 / 계획 검토 / 승인 대기 / 설치 중 / 구조 검증 / 완료`

운영 중 기존 프로젝트는 상세 감사에 `EXISTING_PROJECT_MIGRATION_AUDIT.md`를 사용한다. 사용자 승인 전 대규모 삭제·이동·통합·강제 개명을 수행하지 않는다.

## 1. 목적

- 사용자가 반복해서 기억하지 않아도 되는 규칙:
- 새 GPT·Codex가 복원해야 하는 상태:
- 이번 설치가 해결할 현재 문제:
- 이번 설치에서 해결하지 않는 범위:
- 보호할 프로젝트 고유 결정·자산·경로:

## 11. 완료 기준

- [ ] GPT와 Codex의 최초 읽기 순서가 같다.
- [ ] 질문별 현행 책임 원본이 하나다.
- [ ] 작업·제품 개발 게이트가 증거와 연결된다.
- [ ] L1 이상 작업에 주 책임·영향 분야·변경 유형이 있다.
- [ ] 변경 유형별 필수 갱신 문서가 판정된다.
- [ ] Foundation·분야 스킬과 Learning Log가 연결된다.
- [ ] 이미지에 캐노니컬 경로·상태·채택 범위가 있다.
- [ ] 기존 승인 이미지 임의 교체 금지가 명시돼 있다.
- [ ] 분야 PDF가 전체 과정·승인 이미지·현재 상태를 포함한다.
- [ ] PDF 최신성 파이프라인 상태가 정직하게 표시된다.
- [ ] PR에서 문서·이미지·스킬·PDF 누락을 검사할 경로가 있다.
- [ ] 승인·구현·검증·미확정·보류가 분리돼 있다.
- [ ] 백업·보류·제거 후보가 기본 컨텍스트에서 제외된다.
- [ ] 변경 전후 고유 정보와 참조가 보존됐다.
- [ ] 새 작업자가 10분 안에 방향·상태·게이트·스킬·PDF·검증을 찾는다.
- [ ] 수행하지 않은 검증을 완료로 표시하지 않았다.
