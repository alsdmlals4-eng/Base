# [스킬명] 학습 기록

- 분야:
- 책임 스킬:
- 마지막 검토일:
- 기준 커밋:

> 장문 도구 로그 전체를 보관하지 않는다. 재현에 필요한 조건, 결과, 실패 원인과 스킬 계약 변경만 남긴다.

## 지식 상태 정의

- `관찰`: 한 번 발견한 현상
- `가설`: 추가 검증이 필요한 해결 방법
- `패턴`: 프로젝트 안에서 반복된 경향
- `검증`: 여러 조건에서 재현된 안정적 방법
- `승격 후보`: Base 공용 원리로 검토할 가치가 있음

## 실행 기록

| 날짜·PR | 작업·범위 | 적용 스킬 기준 | 검증 | 성공 조건 | 실패·예외 | 사용자 피드백 | 스킬 변경 | 지식 상태 |
|---|---|---|---|---|---|---|---|---|
|  |  |  |  |  |  |  |  |  |

```md

- 실행한 검증:
- 실행하지 못한 검증:
- 성공한 조건:
- 실패·예외·재현 조건:
- 원인:
- 사용자 피드백:
- 수정한 스킬 계약:
- 재발 방지 기준:
- 현재 지식 상태:
- 다음 검증 조건:
- 프로젝트 전용으로 유지할 내용:
- Base 환류 후보:
```

## 반복 패턴 검수

| 관찰·가설 | 반복 횟수·조건 | 반례 | 현재 판정 | 다음 검증 |
|---|---|---|---|---|
|  |  |  |  |  |

## Base 환류 검수

- [ ] 프로젝트 고유 세계관·수치·경로·승인 자산을 제거했다.
- [ ] 여러 조건 또는 프로젝트에서 재사용 가치가 있다.
- [ ] 기존 Base method·skill·template·case와 중복을 확인했다.
- [ ] 적용 조건과 사용하지 않는 조건이 있다.
- [ ] 실패 조건과 검증 방법이 있다.
- [ ] 한 번의 성공을 검증된 공용 규칙으로 과장하지 않았다.
