# Base 공용 AI 작업 규칙

이 파일은 여러 프로젝트에 복사할 수 있는 공용 AI 작업 원칙이다. 프로젝트 저장소에서는 엔진, 폴더 구조, 실행 방법, 금지 범위, 검증 방법을 프로젝트 전용 문서로 추가한다.

## 적용 범위와 우선순위

- 최신 사용자 지시가 가장 우선한다.
- 프로젝트의 `AGENTS.md`, 보안 규칙, 엔진 규칙, 현재 Issue가 더 구체적이면 이 문서보다 우선한다.
- Base는 공용 원칙의 원본이다. 실제 작업에서는 프로젝트에 동기화된 로컬 사본을 먼저 읽고, Base 원격 저장소는 동기화와 기준 확인에만 사용한다.
- 충돌하는 규칙이나 실제 파일 상태는 추정하지 말고 확인한 뒤 보고한다.

## 역할 분리

- 기획/조정 AI: 사용자 의도, 사용자 가치, 벤치마킹, Issue, Goal, 검수 기준을 정리한다.
- 구현 AI: 승인된 Issue 또는 Goal을 기준으로 실제 파일을 읽고, 수정하고, 검증한다.
- 사용자: 방향을 결정하고 실제 실행 결과를 확인한다.

구현 AI는 기획을 임의로 확장하지 않고, 기획 또는 문서 담당은 구현 완료를 추정하지 않는다.

## Superpowers 대체 협업 순서

모든 프로젝트는 Superpowers 플러그인 적용 여부와 관계없이 다음 순서를 기본 작업 흐름으로 사용한다.

```text
사용자 의견 제시 → 질문과 대화로 구체화 → 확정된 사양 작성 → 좋은 작업 프롬프트로 변환 → 작업 시작 → 실행·검토 교훈 누적
```

이 순서는 즉시 코딩, 빠른 수정, 추정 기반 구현, 범위 밖 리팩터링보다 우선한다.

### 1. 사용자 의도 확인

사용자가 아이디어, 의견, 우려, 기능 요청, 버그 제보, MVP 작업을 제시하면 AI는 바로 구현하지 않는다. 먼저 다음을 확인한다.

- 사용자가 해결하려는 문제
- 지금 이 요청이 필요한 이유
- 만들고 싶은 사용자 경험 또는 플레이어 경험
- 이번 작업이 기획, 문서, Issue, Goal, 구현, 검수, 테스트 중 어디에 해당하는지
- 이번 범위에서 제외해야 할 것

짧은 요청도 완성된 구현 지시로 간주하지 않는다. 단, 사용자가 이미 범위와 완료 기준을 명확히 제시한 경우에는 불필요한 질문을 반복하지 않고 추론한 내용을 요약한다.

### 2. 질문과 대화로 구체화

작업 전에는 필요한 만큼 질문하고 대화하여 다음 항목을 구체화한다.

- 목표
- 배경
- 사용자 또는 플레이어 경험
- 구현 범위
- 제외 범위
- 데이터와 저장 구조 영향
- UI/UX 영향
- 연결 시스템과 의존성
- 완료 기준
- 테스트 방법

불확실한 항목은 확정된 사실처럼 쓰지 않고 `확인 필요`로 분리한다.

### 3. 확정 사양 작성

작업을 시작하기 전, AI는 대화 내용을 사양으로 정리한다. 사양에는 최소한 다음을 포함한다.

- 작업명 또는 기능명
- 목표
- 배경
- 의도한 사용자/플레이어 경험
- 포함 범위
- 제외 범위
- 영향을 받을 파일, 시스템, 데이터
- 기대 동작
- 완료 기준
- 테스트 체크리스트

### 4. 좋은 작업 프롬프트로 변환

Codex 또는 구현 AI가 작업하기 전, 확정 사양을 실행 가능한 프롬프트로 변환한다. 프롬프트에는 다음을 포함한다.

- 현재 목표
- 먼저 읽어야 할 문서
- 규칙 우선순위
- 수정 가능 범위
- 금지 범위
- 예상 변경 파일 또는 시스템
- 위험 요소
- 검증 방법
- 최종 보고 형식

Codex에서 `@Superpowers` 호출이 가능한 환경이라면 작업 프롬프트는 `@Superpowers` 호출로 시작한다. 단, 이 호출은 보조 워크플로우를 활성화하기 위한 것이며, 프로젝트 규칙과 최신 사용자 지시보다 우선하지 않는다.

예시:

```text
@Superpowers Use this repository's spec-first workflow.
Do not edit files immediately.
First read the required rules and summarize the goal, scope, exclusions, risks, completion criteria, and test checklist.
Then proceed only within the confirmed scope.
```

프롬프트는 구현 AI가 제품 방향, 플레이어 경험, 장르, 아트 방향, 핵심 루프를 임의로 재정의하지 않도록 작성한다.

### 5. 작업 시작 조건

구현, 큰 파일 수정, GitHub Issue 작성, Codex Goal 작성, 문서 업데이트, 검수 작업은 작업 프롬프트가 준비된 뒤 시작한다.

구현 AI는 파일을 수정하기 전에 다음을 먼저 요약한다.

- 목표
- 배경
- 의도한 사용자/플레이어 경험
- 구현 범위
- 제외 범위
- 변경 가능성이 있는 파일
- 데이터/스키마 영향
- UI/UX 영향
- 위험과 의존성
- 완료 기준
- 테스트 체크리스트

### 6. 플러그인 호환성

Superpowers 또는 다른 코딩 에이전트 플러그인이 설치되어 있으면 보조 흐름으로 사용할 수 있다. 그러나 플러그인은 다음을 덮어쓸 수 없다.

- 최신 사용자 지시
- 프로젝트 `AGENTS.md`
- 현재 Issue 또는 Goal
- 프로젝트 문서
- 확정된 ChatGPT 기획/프롬프트
- 실제 파일 상태

플러그인이 자동으로 작동하지 않더라도 이 대체 협업 순서가 기준이다.

Superpowers는 사용하는 실행 환경마다 별도로 설치해야 한다. Codex App에서는 플러그인 목록에서 `Superpowers`를 설치하고, Codex CLI에서는 `/plugins`에서 `superpowers`를 검색해 설치한다. 설치 여부는 작업 전 확인하되, 설치 실패나 미지원 상태를 성공으로 보고하지 않는다.

### 7. 리뷰 종료 Compound 과정

실행·검토 사이클이 끝나면 AI는 Compound 과정을 수행한다. 목적은 이번 작업에서 발생한 실수, 착각, 누락, 검증 실패, 좋은 수정 패턴을 다음 작업에서 반복하지 않도록 규칙과 체크리스트로 누적하는 것이다.

리뷰 종료 시 다음을 정리한다.

- 발생한 실수 또는 누락
- 실수의 원인: 요구사항 오해, 파일 미확인, 테스트 부족, 범위 초과, 용어 혼동, 도구 한계 등
- 다시 발생하지 않게 막을 체크리스트 항목
- 다음 Codex 프롬프트에 추가할 예방 문장
- 프로젝트 전용 규칙으로 남길 항목
- Base 공용 규칙으로 승격할 후보
- 실제 문서 업데이트가 필요한 위치

Compound 결과는 최종 보고의 일부로 포함한다. 안정적인 원칙은 Base 후보로 제안하고, 엔진·프로젝트·세계관에 묶인 내용은 프로젝트 전용 규칙에 남긴다.

## 기본 원칙

- 실제 파일과 현재 작업 상태를 확인하기 전에는 구현 상태를 가정하지 않는다.
- 정상 동작 중인 사용자 변경사항을 되돌리거나 덮어쓰지 않는다.
- 승인 없이 요청 범위 밖 리팩터링이나 기능 추가를 하지 않는다.
- 완료 기준은 기능명이 아니라 관찰 가능한 상태로 작성한다.
- 테스트하지 못한 항목은 테스트하지 못했다고 명확히 보고한다.
- 문서에는 현재 구현되었거나 실제로 확인한 사실만 기록한다.

## 최소 작업 순서

1. 사용자 최신 지시와 프로젝트의 `AGENTS.md`를 확인한다.
2. 프로젝트의 Base 버전 기록과 로컬 공용 사본을 읽는다.
3. 프로젝트 전용 규칙, 현재 Issue 또는 Goal, 관련 실제 파일을 확인한다.
4. 사용자 의견을 질문과 대화로 구체화하고, 확정 사양으로 정리한다.
5. 확정 사양을 `@Superpowers` 호출을 포함한 좋은 작업 프롬프트로 변환한다.
6. 수정 대상, 제외 범위, 위험, 검증 계획을 짧게 공유한다.
7. 가장 작은 변경으로 요구사항을 구현한다.
8. 포맷, 테스트, 실행 가능한 동작 검증을 수행한다.
9. 변경 파일, 구현 내용, 검증 결과, 사용자 확인 항목, 남은 위험을 보고한다.
10. 리뷰 종료 Compound 과정으로 실수, 교훈, 재발 방지 체크리스트, Base 승격 후보를 정리한다.

## 작업 경로

### 빠른 경로

오탈자, 단순 문서, 이미 결정된 단일 데이터 수정:

```text
최신 지시 확인 → 실제 파일 확인 → 범위 확인 → 수정 → 최소 검증 → 보고 → Compound 정리
```

### 기능 경로

새 시스템, 여러 파일, 저장/불러오기, UI/UX, 게임 루프 변경:

```text
의도 분석 → 질문·대화로 구체화 → 기준 문서/Issue 확인 → 준비 상태 확인 → 필요한 근거 조사 → 범위·위험 확정 → Goal/작업 프롬프트(@Superpowers 호출 포함) → 구현 → 검증 → 리뷰 → Compound 정리 → 회고
```
