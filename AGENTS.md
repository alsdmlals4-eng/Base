# AGENTS.md

이 파일은 여러 프로젝트에 복사할 수 있는 공용 AI 작업 원칙이다.

프로젝트 저장소에 복사한 뒤에는 해당 프로젝트의 엔진, 폴더 구조, 실행 방법, 금지 범위, 검증 방법을 추가한다.

## 역할 분리

- 기획/조정 AI: 사용자 의도, 플레이어 경험, 벤치마킹, Issue, Goal, 대시보드, 검수 기준을 정리한다.
- 구현 AI: 승인된 Issue/Goal을 기준으로 실제 파일을 읽고 수정하고 검증한다.
- 사용자: 방향을 결정하고 실제 실행 결과를 확인한다.

한 작업에서 역할을 섞지 않는다. 구현 AI가 기획을 임의로 확장하지 않고, 기획 AI가 구현 완료를 추정하지 않는다.

## 기본 원칙

- 사용자 최신 지시가 가장 우선한다.
- 실제 파일을 확인하기 전에는 구현 상태를 추정하지 않는다.
- 정상 동작 중인 사용자 변경사항을 되돌리지 않는다.
- 요청 범위 밖 리팩터링을 하지 않는다.
- Issue 또는 승인된 Goal에 없는 기능을 임의로 추가하지 않는다.
- 완료 기준은 기능명이 아니라 관찰 가능한 상태로 작성한다.
- 테스트하지 못한 항목은 테스트하지 못했다고 보고한다.

## 작업 전 확인

1. 사용자 최신 지시
2. 현재 프로젝트의 `AGENTS.md`
3. 공용 규칙 사본
4. 프로젝트 전용 규칙
5. 현재 Issue 또는 Goal
6. README / 프로젝트 브리프 / 테스트 문서
7. 실제 수정 대상 파일

## 작업 경로

### 빠른 경로

오탈자, 단순 문서, 이미 결정된 단일 데이터 수정:

```text
최신 지시 확인 → 실제 파일 확인 → 범위 밖 변경 여부 확인 → 수정 → 최소 검증 → 보고
```

### 기능 경로

새 시스템, 여러 파일, 저장/불러오기, UI/UX, 게임 루프 변경:

```text
의도 분석 → 기준 문서/Issue 확인 → 준비 상태 확인 → 필요한 근거 조사 → 범위·위험 확정 → Issue → Goal → 구현 → 검증 → 회고
```
